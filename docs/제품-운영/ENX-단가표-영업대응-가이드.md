## 1. 기본 원칙

> **이 세 줄을 항상 머릿속에 넣고 통화한다.**

1. **먼저 가치를 말하고, 숫자는 나중에 꺼낸다.**  
   "특허 출원된 국내 유일 잘피 종자 살포 관제 SaaS입니다" → 그 다음 요금.

2. **단가를 낮추는 것보다 범위를 좁히는 게 낫다.**  
   "그 금액이면 이 기능만 됩니다"가 "그럼 깎아드릴게요"보다 훨씬 유리하다.

3. **공짜 작업은 없다. 단, 리스크를 낮춰주는 대안은 있다.**  
   무조건 거절이 아니라 **유료 PoC(소규모 시범)**로 유도한다.

4. **월 구독 ≠ 현장·개발 비용.**  
   **월 50만 원(선박 1척)** 은 SaaS 이용료. **초기 세팅·PLC 기본/특수설치·출장·A/S 수정·추가 코딩** 은 **항상 별도** 견적·청구한다. **특수설치(기계·타이머 제어)** 는 무조건 **협의**. 통화에서 "월 얼마"만 말하고 세팅·출장을 숨기지 않는다.

## 2. 요금제 체계

### 월 구독 (핵심)

| 항목 | 금액 |
|------|------|
| **기본 월 구독** | **월 50만 원** (선박 **1척**) |
| **선박 추가** | **월 20만 원/척** (2척째부터) |
| **STANDARD (선택)** | **월 100만 원** — 최대 3척, PDF·카카오·펌웨어 월 1회 등 |
| **PREMIUM (선택)** | **월 200만 원** — 다척·맞춤 UI·전용 환경·우선 지원 (협의) |

> **대외 1페이지:** [구독-요금제-소개페이지.md](./구독-요금제-소개페이지.md)

### 구독에 넣지 않는 것 (별도 청구)

| 항목 | 대략 단가 | 비고 |
|------|-----------|------|
| 초기 세팅 (원격) | 50~130만 원 (1회) | 범위 좁을수록 하한 |
| **PLC 기본설치** | **50~80만 원/척** (1회) | 충전형 PLC(함체 재작)·**웹 서버 수신** 연동 |
| **PLC 특수설치** | **협의** | 기계부·타이머·비표준 PLC 제어 — §2-1 |
| 현장 출장 | 기술료 30만 원~/일 + 실비 | §5 참고 |
| A/S·수정·요구 변경 | **유상 견적** | 제품 결함만 무상 협의 |
| 추가 코딩·API 연동 | §4 표 | 착수금 50% 등 |

### 2-1. 현장 PLC 설치 — 기본설치 / 특수설치

> 통화·견적에서 **「PLC 설치」** 를 이 둘로 나눠 말한다. 섞어서 한 가격으로 부르지 않는다.

#### 기본설치 (표준 · 1척당 50~80만 원 권장)

| 항목 | 내용 |
|------|------|
| **장비** | **함체에 재작(개조)된 충전형 PLC** (ENX 표준) |
| **역할** | 살포·GPS·상태 등을 **관제 웹 서버·DB로 수신** (Telemetry / POST / LTE 등 **표준 경로**) |
| **포함** | PLC·통신·펌웨어(또는 동등 단말)·**1척 관제 연결 완료** |
| **미포함** | 선박 **기계 본체** 릴·펌프·타이머 릴레이 **구동 로직**·비표준 배선 |
| **출장** | 원격 가능 시 출장 0 · 현장 필요 시 **§5 출장비 별도** |

**통화 한 줄:**  
「기본은 **충전 PLC 달고 관제 서버로 신호 받는 것**까지예요. **월 50만은 그다음 쓰는 비용**이고, 기본설치는 **1회 50~80만** 잡으시면 됩니다.」

#### 특수설치 (협의 · 견적 필수)

| 항목 | 내용 |
|------|------|
| **대상** | **기계 부** — 타이머·릴레이·펌프·밸브 등을 **특수 방식·시퀀스**로 묶는 **PLC 프로그램·배선** |
| **예시** | 다단 타이머, 구형 장비 맞춤, 살포기계 **직접 기동/정지** 연동, 비표준 안전 인터록 |
| **절차** | 현장 사진·회로·장비명 → **기술 검토 3일 내 견적** → 착수금 50%(100만 원 이상 시 §4 원칙) |
| **금액** | **협의** (통상 100만 원~, 공수·위험도에 따라) |

**통화 한 줄:**  
「선박 **기계를 특수하게 돌려야** 하면 **특수설치**라서, **기본 견적에 넣지 않고** 현장 보고 **따로 견적** 드립니다.」

**판단 기준 (내부):**

- 「관제에만 보이면 된다」→ **기본설치**만 제시  
- 「PLC가 펌프/릴/타이머를 직접 제어해야 한다」→ **특수설치** + 기본설치(수신) **분리 견적**

## 3. 초기 세팅 비용 시나리오

> 구독료와 **별도**로 최초 1회 발생하는 비용입니다.

### 시나리오 A — 원격 세팅 (클라이언트가 장치 보유, 원격으로 코딩)

| 항목 | 금액 | 비고 |
|------|------|------|
| 원격 설정비 | **30만 원** (1회) | 장치 정상 작동 전제, 1~2시간 원격 지원 |
| 하드웨어 코딩 (아두이노 펌웨어) | **80만 원** (1회) | 클라이언트 장치에 ENX 펌웨어 설치 |
| 클라우드 계정·도메인 설정 | **20만 원** (1회) | 기관 전용 URL 설정 포함 |
| **소계** | **130만 원** | |

> **주의:** 클라이언트 장치가 ENX 권장 사양(ESP32·NEO-M8N GPS 등)이 아닐 경우  
> 호환성 검토비 **50만 원** 추가 가능. 사전에 장치 모델명 확인 필수.

## 4. 요구사항 추가 시 단가 체계

> 기본 플랜 외에 기관이 "이것도 해줄 수 있어요?"라고 물을 때 적용.

|-------------|----------|------|----------|
| 선박 1척 추가 (BASIC·STANDARD 초과분) | 소 | **월 20만 원/척** | 구독료에 합산 |
| **PLC 기본설치** (1척) | 중 | **50~80만 원** (1회) | §2-1 · 웹 서버 수신까지 |
| **PLC 특수설치** | 대 | **협의** | §2-1 · 기계·타이머 제어 |
| 기관 로고·색상 커스터마이징 | 소 | **50만 원** (1회) | 초기 세팅 시 |
| 전용 관리자 계정 추가 (5개 초과) | 소 | **월 5만 원/계정** | 구독료에 합산 |
| 살포 구역 폴리곤 직접 편집 기능 | 중 | **150만 원** (1회) | 개발 완료 후 |
| 타 정부 시스템 API 연동 (단방향) | 중 | **200만 원** (1회) | 개발 완료 후 |
| 타 정부 시스템 API 연동 (양방향) | 대 | **400만 원~** (협의) | 착수금 50% + 완료 50% |
| 살포 실적 자동 공문 생성 | 중 | **180만 원** (1회) | 개발 완료 후 |
| 다국어 지원 (영문 추가) | 중 | **200만 원** (1회) | 개발 완료 후 |
| 드론 연동 모듈 추가 | 대 | **500만 원~** (협의) | 착수금 50% + 완료 50% |
| 오프라인(Wi-Fi 단절) 모드 | 대 | **350만 원~** (협의) | 착수금 50% + 완료 50% |

> **원칙:** 100만 원 이상 개발 건은 **착수금 50% 선수령** 후 착수합니다.  
> 금액 미정 항목은 **"기술 검토 후 3일 내 견적 드리겠다"** 고 답하고 무조건 즉답하지 않습니다.

## 5. A/S·현장 방문·긴급 출동

| 상황 | 비용 |
|------|------|

> **통화 중 팁:** "현장 가는 건 추가 비용이에요, 원격으로 해결되는 건 포함이에요" 라고 명확히 말합니다.

## 6. 계약 기간별 할인 구조

## 8. "얼마에요?" 질문 대응 스크립트

### 8-1. 즉답을 피하고 범위를 좁히는 법

> 상대방: "관제 시스템 쓰면 한 달에 얼마예요?"

**나쁜 답변 (하지 말 것):**  
"아, 한 달에 100만 원이요." → 싸게 불렀다가 나중에 기능 추가 때 분쟁.

**좋은 답변 (이렇게):**  
"선박 몇 척이신지, 단말이 이미 있으신지에 따라 달라요. **월 구독은 선박 1척 기준 50만 원**이고, **처음 붙이는 연결·세팅·출장은 따로** 잡습니다. 그래서 1년 총액을 말씀드릴 때는 월 구독이랑 1회성을 나눠서 보여 드릴게요. 여쭤봐도 될까요?"

→ 정보를 먼저 얻고, 플랜을 맞춰서 제시합니다.

---

### 8-2. 플랜 제시 순서

1. **BASIC부터 제시** — **월 50만 원(1척)** + **세팅·연결 1회성** (예: 1년 700만 원대)
2. 선박 2~3척·보고 강화 → **STANDARD** (월 100만 원) 또는 **50만 + 척당 20만**
3. 다기관·맞춤 → **PREMIUM** (월 200만 원~) — 먼저 제시하지 않음
4. **출장·추가 코딩은 구독에 포함 안 된다**고 반드시 말함

---

### 8-3. "비싸다" 반응 대응

> 상대방: "그거 너무 비싼 거 아닌가요?"

**이렇게:**  
"기존에 시스템 새로 구축하시면 SI 예산이 최소 3억~5억 원이고, 구축 후 유지보수도 따로 들잖아요. 저희는 초기 구축비 없이 월 구독이라, 1년 써보시고 맞지 않으면 해지하셔도 됩니다. 3억짜리 시스템 발주하고 현장에 방치되는 것보다 훨씬 안전한 거죠."

→ **비교 기준을 SI 구축비로 올립니다.**

## 9. "먼저 만들어줘, 예산 나오면 줄게" 대응 전략

> 이런 요청이 가장 위험합니다. 아래 3단계로 대응합니다.

### 9-1. 왜 이 요청이 위험한가 (내 머릿속용)

- 예산은 "나오면"이 아니라 **기관 내부 승인 프로세스**에 달려 있어 확신 불가
- 완성물이 넘어간 순간 협상력 제로
- 개발 중 요구사항이 계속 늘어나도 막을 방법 없음
- 법적 계약 없이 구두 약속은 분쟁 시 증거 없음

---

### 9-2. 1차 대응 — 공감 + 대안 제시

> 상대방: "아직 예산이 확정이 안 됐는데, 먼저 시스템 좀 만들어주시면 나중에 예산 나오면 계약할게요."

**이렇게:**  
"그 상황 충분히 이해해요. 저도 기관 예산 사이클 알거든요. 근데 저희가 선 작업하고 나중에 받는 구조는 제 회사 운영이 안 되는 방식이라서요. 대신에 이렇게 하시면 어떨까요. 지금 당장 전체 구축은 아니고, 현재 완성된 시스템을 **3개월만 써보시는 PoC**로 시작하시는 거예요. **세팅·연결 한 번**이랑 **월 40만 원 구독**이면 3개월에 **230만 원 전후**인데, 소액 용역·기술 자문 예산으로 집행되는 경우가 많아요."

---

### 9-3. 2차 대응 — 끝까지 "무조건 먼저"를 요구할 때

> 상대방: "그래도 뭔가 보여줘야 윗선에서 예산을 잡아줄 것 같아서요."

**이렇게:**  
"데모 링크랑 1페이지 소개서는 지금 바로 드릴 수 있어요. 특허 출원 문서도 드릴 수 있고요. 그걸로 내부 품의 올리시면 윗선 설득에 충분하지 않을까요? 반면에 제가 기관 맞춤으로 미리 만들어드리면, 그건 이미 '개발 용역'이 시작된 거라서 계약 없이는 할 수 없어요. 이 부분은 제가 사업자니까 양해 부탁드려요."

---

### 9-4. 3차 대응 — 거절해야 할 때의 마무리

> 상대방이 계속 무료 작업을 요구할 경우.

**이렇게:**  
"제 상황을 솔직하게 말씀드리면, 저도 기술로 먹고 사는 사람이라서 선 작업 후 대금 방식은 지속 가능하지 않아요. 다만 나중에 예산이 확정되면 꼭 연락 주세요. 그때 바로 시작할 수 있도록 준비하고 있을게요."

→ **관계는 유지하되 작업은 하지 않습니다.**  
→ 번호 끊지 말고, 예산 확정 후 연락 오면 그때 계약.

---

### 9-5. 정말 가능성 있는 기관인데 관계를 이어가고 싶을 때

조건부 협력 제안 (이 경우만 예외적으로 검토):

1. **MOU(업무협약서) 먼저 체결** → 상호 신뢰 구조 만들기
2. **시스템 사용 기간 중 실비(서버비)만 청구** → 개발 공수는 미청구, 단 MOU에 예산 확정 시 정식 계약 명시
3. **기간 제한 명시** — "6개월 내 예산 미집행 시 MOU 자동 소멸" 문구 포함

> **주의:** 이 예외는 기관의 실명·담당자 정보·MOU 체결이 모두 갖춰진 경우만 검토합니다.  
> 전화 통화 단계에서 구두 약속만 있는 경우엔 절대 적용하지 않습니다.
