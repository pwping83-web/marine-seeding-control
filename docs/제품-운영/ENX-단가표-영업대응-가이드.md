## 1. 기본 원칙

> **이 세 줄을 항상 머릿속에 넣고 통화한다.**

1. **먼저 가치를 말하고, 숫자는 나중에 꺼낸다.**  
   "특허 출원된 국내 유일 잘피 종자 살포 관제 SaaS입니다" → 그 다음 요금.

2. **단가를 낮추는 것보다 범위를 좁히는 게 낫다.**  
   "그 금액이면 이 기능만 됩니다"가 "그럼 깎아드릴게요"보다 훨씬 유리하다.

3. **공짜 작업은 없다. 단, 리스크를 낮춰주는 대안은 있다.**  
   무조건 거절이 아니라 **유료 PoC(소규모 시범)**로 유도한다.

## 2. 요금제 체계

| 구분 | **BASIC** | **STANDARD** | **PREMIUM** |
|------|-----------|--------------|-------------|
| **대상** | 단순 실적 확인이 필요한 소규모 기관 | 선박 1척 이상 운영, 현장 데이터 수집 | 다기관 통합, 커스터마이징, 우선 지원 |
| **월 구독료** | **월 120만 원** | **월 250만 원** | **월 450만 원** |
| **연 환산** | 1,440만 원/년 | 3,000만 원/년 | 5,400만 원/년 |
| **관리 선박 수** | 1척 | 최대 3척 | 무제한 |
| **웹 관제 대시보드** | ✅ | ✅ | ✅ |
| **살포 구역 면적(ha) 자동 산출** | ✅ | ✅ | ✅ |
| **기상청 데이터 연동** | ✅ | ✅ | ✅ |
| **항로 생성 (장애물 회피)** | ✅ | ✅ | ✅ |
| **GPS 실시간 위치 연동** | ❌ | ✅ | ✅ |
| **아두이노·Telemetry 서버 연동** | ❌ | ✅ | ✅ |
| **살포 이력 CSV/보고서 자동 출력** | 기본 | ✅ | ✅ |
| **기관 맞춤 UI·로고 커스터마이징** | ❌ | ❌ | ✅ |
| **전용 서버 환경 분리** | ❌ | ❌ | ✅ |
| **전화 우선 지원** | 이메일만 | 이메일+카카오 | **전화 우선** |
| **월 업데이트 보고서** | ❌ | ❌ | ✅ |

> PREMIUM을 먼저 제시하지 않습니다 (너무 크게 느껴지면 대화 끊김).

## 8. "얼마에요?" 질문 대응 스크립트

### 8-1. 즉답을 피하고 범위를 좁히는 법

> 상대방: "관제 시스템 쓰면 한 달에 얼마예요?"

**나쁜 답변 (하지 말 것):**  
"아, 한 달에 100만 원이요." → 싸게 불렀다가 나중에 기능 추가 때 분쟁.

**좋은 답변 (이렇게):**  
"선박 몇 척 운영하시고, 현장 GPS 장치가 이미 있으신가요? 그에 따라 월 120만 원부터 시작해서 구성에 따라 달라지거든요. 여쭤봐도 될까요?"

→ 정보를 먼저 얻고, 플랜을 맞춰서 제시합니다.

---

### 8-2. 플랜 제시 순서

1. **STANDARD부터 제시** (월 250만 원)
2. 예산 부족 반응이면 → **BASIC 제시** (월 120만 원)
3. "더 많이 쓰고 싶다" 반응이면 → **PREMIUM 제시** (월 450만 원)
4. 절대로 PREMIUM을 먼저 제시하지 않습니다.

---

### 8-3. "비싸다" 반응 대응

> 상대방: "그거 너무 비싼 거 아닌가요?"

**이렇게:**  
"기존에 시스템 새로 구축하시면 SI 예산이 최소 3억~5억 원이고, 구축 후 유지보수도 따로 들잖아요. 저희는 초기 구축비 없이 월 구독이라, 1년 써보시고 맞지 않으면 해지하셔도 됩니다. 3억짜리 시스템 발주하고 현장에 방치되는 것보다 훨씬 안전한 거죠."

→ **비교 기준을 SI 구축비로 올립니다.**

---

### 8-4. "데모 먼저 봐도 되나요?"

## 9. "먼저 만들어줘, 예산 나오면 줄게" 대응 전략

> 이런 요청이 가장 위험합니다. 아래 3단계로 대응합니다.

### 9-1. 왜 이 요청이 위험한가 (내 머릿속용)

- 예산은 "나오면"이 아니라 **기관 내부 승인 프로세스**에 달려 있어 확신 불가
- 완성물이 넘어간 순간 협상력 제로
- 개발 중 요구사항이 계속 늘어나도 막을 방법 없음
- 법적 계약 없이 구두 약속은 분쟁 시 증거 없음

---

### 9-2. 1차 대응 — 공감 + 대안 제시

> 상대방: "아직 예산이 확정이 안 됐는데, 먼저 시스템 좀 만들어주시면 나중에 예산 나오면 계약할게요."

**이렇게:**  
"그 상황 충분히 이해해요. 저도 기관 예산 사이클 알거든요. 근데 저희가 선 작업하고 나중에 받는 구조는 제 회사 운영이 안 되는 방식이라서요. 대신에 이렇게 하시면 어떨까요. 지금 당장 전체 구축은 아니고, 현재 완성된 시스템을 **3개월만 써보시는 PoC**로 시작하시는 거예요. 비용은 초기 세팅 130만 원에 월 200만 원이니까 3개월에 730만 원인데, 이 정도는 소액 용역이나 기술 자문 예산으로 집행 가능한 경우가 많아요."

---

### 9-3. 2차 대응 — 끝까지 "무조건 먼저"를 요구할 때

> 상대방: "그래도 뭔가 보여줘야 윗선에서 예산을 잡아줄 것 같아서요."

**이렇게:**  
"데모 링크랑 1페이지 소개서는 지금 바로 드릴 수 있어요. 특허 출원 문서도 드릴 수 있고요. 그걸로 내부 품의 올리시면 윗선 설득에 충분하지 않을까요? 반면에 제가 기관 맞춤으로 미리 만들어드리면, 그건 이미 '개발 용역'이 시작된 거라서 계약 없이는 할 수 없어요. 이 부분은 제가 사업자니까 양해 부탁드려요."

---

### 9-4. 3차 대응 — 거절해야 할 때의 마무리

> 상대방이 계속 무료 작업을 요구할 경우.

**이렇게:**  
"제 상황을 솔직하게 말씀드리면, 저도 기술로 먹고 사는 사람이라서 선 작업 후 대금 방식은 지속 가능하지 않아요. 다만 나중에 예산이 확정되면 꼭 연락 주세요. 그때 바로 시작할 수 있도록 준비하고 있을게요."

→ **관계는 유지하되 작업은 하지 않습니다.**  
→ 번호 끊지 말고, 예산 확정 후 연락 오면 그때 계약.

---

### 9-5. 정말 가능성 있는 기관인데 관계를 이어가고 싶을 때

조건부 협력 제안 (이 경우만 예외적으로 검토):

1. **MOU(업무협약서) 먼저 체결** → 상호 신뢰 구조 만들기
2. **시스템 사용 기간 중 실비(서버비)만 청구** → 개발 공수는 미청구, 단 MOU에 예산 확정 시 정식 계약 명시
3. **기간 제한 명시** — "6개월 내 예산 미집행 시 MOU 자동 소멸" 문구 포함

> **주의:** 이 예외는 기관의 실명·담당자 정보·MOU 체결이 모두 갖춰진 경우만 검토합니다.  
> 전화 통화 단계에서 구두 약속만 있는 경우엔 절대 적용하지 않습니다.
