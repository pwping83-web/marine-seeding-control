# 해양 종자 살포 관제 시스템 — 개요 및 작동 방식

## 문서 목적

본 문서는 현재 저장소에 구현된 **해양 종자 살포 관제** 웹 화면의 **컨셉**, **동작 흐름**, **기술 구성**, **백엔드 유무**를 한글로 정리합니다. 실제 관공서·내부망 연동 전 **시연·납품용 UI 프로토타입** 기준입니다.

## 시스템 컨셉

- **대상**: 해조류 복원 등 **종자 살포** 작업을 실시간에 가깝게 모니터링하는 관제 데스크톱 UI.
- **톤**: 어두운 남색 계열. 좌측 사이드바에 브랜드·요약·**AI 기상 안전**·선박 위치·신호·이력을 쌓고, 본문 상단에 **AI 요약 자막**(한 바 스크롤 후 30초 정지 반복)을 둡니다.
- **지도**: **Leaflet** 기반 타일 지도(`MarineLeafletMap`). 선박·항적·살포 점·(선택) A* 항로·목적지 마커 표시. **지도 클릭**으로 목적지 지정 가능.
- **데이터**: 기본은 브라우저 내 **시연·시뮬레이션**. **Supabase** 환경 변수가 설정되면 살포·궤적·선박 명령(`ship_command_logs` 등) 연동 가능. 새로고침 시 로컬 목 데이터는 초기화되는 동작이 기본입니다.

## 백엔드·외부 연동 현황

| 항목 | 현재 상태 |
|------|-----------|
| **자체 백엔드 서버** | **없음.** Node/Express 등 별도 API 서버는 저장소에 포함되지 않습니다. |
| **Supabase** | **선택.** `VITE_SUPABASE_*` 설정 시 Edge·DB와 연동(궤적·살포·명령 로그). 미설정 시 관련 UI는 시연 모드. |
| **기상청 오픈 API** | **선택.** `VITE_KMA_SERVICE_KEY` 등 설정 시 단기·초단기예보 기반 분석. 미설정 시 대시보드용 시뮬 기상. |
| **기상청 중기예보** | **선택(동일 키).** `WorkPlanView`에서 D+3 이후 날짜를 중기 API(`MidFcstInfoService`)로 보강. 미수신·키 없음 시 모의 중기로 합성. `kma-weather.ts`의 `fetchKmaMiddleTermForecast` 등. |
| **AI기상 위험 상황 자동 요약 리포트** | **선택.** `VITE_GROQ_API_KEY` 설정 시 Groq 등으로 기상·위험 요약 문장 생성. |
| **로그인** | 시연용. 클라이언트만으로 접속 처리(실 SSO 없음). **이메일 알림(EmailJS)** 은 코드상 주석 처리됨. |

실서비스 전환 시: 인증(BFF/SSO), 영구 DB, WebSocket/MQTT 푸시 등을 별도 설계합니다.

## 화면 흐름 및 작동 방식

### 1) 로그인 화면 (`LoginPage`)

- 이메일·비밀번호 폼은 있으나 **시연 모드**로 검증 없이 대시보드로 전환 가능.
- 접속 알림 이메일(EmailJS) 연동 코드는 **주석 처리**되어 있습니다.

### 2) 관제 대시보드 (`Dashboard`)

- **좌측 사이드바** (`h-svh`, 상단 고정 블록 + **살포 이력만** 남은 세로를 채우고 목록만 스크롤)
  - **브랜드·모드 탭**: 실시간 관제 / 작업 계획 전환.
  - **요약 3칸**: 금일 살포·누적 건수·속도(노트).
  - **AI 기상 안전**: 안전 등급 띠, 풍향 나침반·풍속·파고 등 요약, `WeatherAIPanel`이 판정·예보 슬롯·Groq 요약(키 있을 때)을 갱신. **표시용 등급**은 `assessEmergency`(즉시 회항 임계)와 단기예보 **첫 시간대** `scoreHourSlot` 판정을 **더 보수적으로 합친 값**(`mergeDisplaySafetyLevel` in `Dashboard.tsx`)으로, 사이드바·AI 자막·지도 하단 타임라인·로컬 작업 권고가 같은 기준을 씁니다.
  - **금일 항적·보고**: `TodayTrackReportModal` — 기간 요약·CSV, **PDF 출력**(브라우저 인쇄 → PDF 저장).
  - **구역 추정 면적**: 살포 점 3개 이상일 때 Convex Hull 근사 **ha** 표시(`seeding-day-eval.ts` / 사이드바).
  - **선박 위치**: 북위·동경·방위.
  - **선박 신호 송신**(2×2): **귀항 명령**, **살포 시작**, **살포 중지**, **위치 보고** — Supabase 연동 시 `ship_command_logs`에 `insertShipCommand`. 살포 시작 시 **「살포 중」** 뱃지에 은은한 호흡 애니메이션.
  - **항로 네비 · 속도 제어**: **항상 펼침.** 목적지·지도 클릭, A*, PID·목표 속도(연동 시).
  - **살포 이력**: 필터 기간 내 전체를 **스크롤 목록**으로 표시, 총 건수·최근 한 줄 요약, 날짜 필터·CSV 불러오기/저장.

- **본문 상단(지도 위)**  
  - **AI 요약 자막**(`AiTicker`): 한 줄이 **오른쪽 밖에서 들어와 왼쪽으로 전부 빠진 뒤** **30초 정지**, 이후 같은 문구를 반복. Groq·긴급 요약 문구는 이 자막에 포함됩니다.

- **우측 지도 영역**
  - **현재 시각** 지도 상단 중앙 오버레이, **시연** ✅⚡🔴 + **고도화** 버튼 우상단 한 줄(살포 색상 안내와 겹치지 않게 배치).
  - **기상 예측 타임라인**(`WeatherTimelineTracker`): 지도 하단, **반투명** 배경으로 지도 비침, 8시간 슬롯·「지금 위험」등 실시간 위험 요약(표시 등급은 위와 동일하게 통합).
  - **살포 색상 안내**: 우측 중간 높이 패널 토글, 범례·설명.
  - **B2G 시연**: 기상 정상/악화/SOS 등 오버레이(일부는 시연 버튼으로 트리거).
  - **고도화 모달**(`VisionRoadmapModal`): 1단계 구현 범위·2~3단계(CCTV·원격조종·**암초 센서·지도 회피**·안전전환) 관공서용 요약.
  - **지도 확대·축소(Leaflet 단계)**: 우측 세로 **＋ / －** 및 지도 컨테이너 **휠**은 `zoomIn(1)`·`zoomOut(1)`과 같이 **현재 뷰 기준 한 단계**씩 조정합니다. **나침반(맞춤)·새로고침(뷰 초기화)·테스트 모드 전환** 등 맞춤을 다시 할 때는 내부 줌 조작 카운터를 함께 초기화합니다. 우측에 **현재 줌 레벨**(예: 「줌 14」)이 표시될 수 있습니다. 구현: `MarineLeafletMap.tsx`의 `MapFitBounds`·`MapZoomByNonce`, `Dashboard.tsx`의 증분 카운터.
  - **항로 길안내 모달**(`TrackNavigationModal`): 「**항로 꼭짓점 찍기**」「**살포 예정 찍기**」를 켜면 **큰 모달이 숨겨지고** 지도만 보이게 한 뒤, 하단 **「항로 설정 끄기」/「살포 예정 설정 끄기」**로 다시 모달을 띄웁니다. 「경로 설정 안내 닫기」는 찍기 모드 중에도 모달·찍기를 함께 종료합니다.
  - **금일 항적 꼭짓점**: 항로 찍기 활성 시 **번호 마커**를 **드래그**하면 좌표가 갱신됩니다. 대시보드가 자주 리렌더될 때도 드래그가 끊기지 않도록 Leaflet `Marker`의 `position`은 **위도·경도 값이 바뀔 때만** 참조가 바뀌게 메모이제이션합니다(`WaypointEditorMarker`).
  - **AI 자막 읽어주기**: 상단 `AiTicker`의 **🔊 읽어주기**는 브라우저 **Web Speech API**로 한국어를 읽으며, OS 음성 중 **자연·신경망 계열**을 우선합니다(`lib/korean-tts.ts`). 자막이 잇달아 바뀔 때는 **끊지 않고** 끝난 뒤 **가장 최근 자막만** 이어 읽고(coalesce), **날씨 급변** 한 줄은 **즉시 덮어** 읽습니다(`AiWeatherJoltBanner`).

- **작업 계획 탭** (`viewMode === "schedule"`)
  - 좌: **AI 살포 계획**(`SeedingPlanPanel`) — 일일 한도, 해역 TOP5, A*, PID 스케치 복사.
  - 우: **작업 계획** 뷰(`WorkPlanView`) — 7일 카드는 단기 합성이 기본이며, **중기예보** 수신 시 해당 날짜·풍파고 등으로 덮어씀. 작업 가능/주의/불가는 **풍속 15 kt 이하·파고 1.5 m 이하·시정 5 km 이상·강수 5 mm 미만** 등 별도 임계(실시간 슬롯의 m/s 점수와는 이중 기준이므로 수치가 다를 수 있음).

- **선박·살포 시연 로직(요약)**
  - 테스트 모드에서 웨이포인트 이동·간헐적 살포 점 추가, LTE/GPS 모드 분기 등 기존 로직 유지.

## 빌드·로컬 실행

```bash
npm install
npm run dev      # 개발 서버 — 기본 http://192.168.45.214:5111/ · 준비되면 기본 브라우저 자동 실행(끄기: VITE_OPEN=0, 열 URL 덮어쓰기: VITE_OPEN_URL)
```

### Cursor / VS Code 안에서 미리보기

- 명령 팔레트(`Ctrl+Shift+P`) → **Tasks: Run Task** → **「심플 브라우저로 앱 열기 (Cursor 창 안)」** 를 실행하면 편집기 내 **심플 브라우저**로 `http://192.168.45.214:5111/` 가 열립니다. (`npm run dev`가 떠 있는 상태여야 합니다.)
- URL은 `.vscode/tasks.json`의 `simpleBrowser.api.open` 인자와 맞춰 두었습니다.

## 기능 개발 예정 (로드맵 요약)

다음은 **현재 코드에 없고** 고도화 모달·단계 계획에 반영된 **기획**입니다.

| 항목 | 방향 |
|------|------|
| **암초·저수심 센서** | 선박 단말에 소나·해저 스캔 등 탐지 센서를 두고, 암초 등 위험물 발견 시 **WGS84 좌표를 자동 기록**합니다. |

| **자동 회피 (목표)** | 누적·공유된 암초 레이어를 다른 관제·선박 사용자가 볼 때, **항로 생성·자율운항**이 기록 좌표를 **우회**하고 접근 시 **경고**하는 흐름을 목표로 합니다. 실해역에서는 **공공 해도·공식 수심 정보와의 검증**이 선행되어야 합니다. |

UI·문구 상세: `VisionRoadmapModal.tsx` 의 **「🪨 암초·회피」** 탭 및 **2·3단계** bullet.

## 정리

| 질문 | 답 |
|------|-----|
| 자체 백엔드가 있나요? | **없습니다.** 선택적으로 **Supabase**만 클라이언트에서 호출합니다. |
| 로그인은 안전한가요? | **시연용**입니다. 실제 SSO·토큰 없음. |
| 데이터는 어디에 저장되나요? | 기본은 **브라우저 메모리·목**. Supabase 연동 시 DB·Edge 경로로 확장. |
| 지도는 무엇인가요? | **Leaflet 타일 지도**입니다. 타일 URL은 구현 설정에 따르고, **＋/－·휠**은 Leaflet과 같이 **한 단계씩** 줌을 바꿉니다. |
| AI 기상은 실제인가요? | 기상청 키·좌표가 있으면 API 기반, 없으면 시뮬. Groq는 자연어 요약용 옵션. |
| 사이드바 안전과 타임라인이 다른가요? | **통합 표시**로 맞춤. 첫 슬롯이 작업 불가여도 긴급 임계만으로는 안전으로 보이던 불일치를 줄였습니다. |

본 문서는 저장소 현재 구현 기준이며, 인프라·인증을 붙일 때마다 절을 갱신하는 것을 권장합니다.

**문서 개정:** v1.9 (2026-05-12) — 지도 줌(Leaflet 단계)·항로 모달(찍기 시 숨김)·꼭짓점 드래그 안정화·AI 읽어주기(coalesce·음성 우선) 반영. 앱 버전 **3.5.2** 기준.
