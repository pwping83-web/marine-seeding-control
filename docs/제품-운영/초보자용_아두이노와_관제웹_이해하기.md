# 아두이노(GPS·LTE)랑 관제 웹 — 아주 쉽게 이해하기

전기·코딩을 잘 모르는 분도 **“누가 뭘 하는지”**만 알면 됩니다.  
비유로 설명하고, 그다음에 **실제로 해야 할 일 순서**를 적었습니다.

---

## 1. 비유로 한 번에 이해하기

### 배 위의 **아두이노** = “현장에서 메모하는 직원”

- **GPS** 달면: “지금 우리 배가 지도상 어디쯤인지” 숫자(위도·경도)를 알 수 있습니다.  
- **LTE** 달면: “휴대폰처럼 인터넷”이 되어서, 육상에 있는 **관제실 컴퓨터(서버)**에게 그 숫자를 **보낼 수** 있습니다.  
- **코딩**이란: “몇 분마다 한 번 보낼지”, “살포 버튼 누르면 그때 좌표를 보낼지” 같은 **규칙을 기기 안에 적어 넣는 것**입니다.

### **우리 관제 웹(브라우저 화면)** = “사무실 큰 지도와 일지”

- 바다에서 온 숫자를 **서버(데이터베이스)**가 받아서 저장합니다.  
- 관제 웹은 그 저장된 내용을 **지도에 점·선으로 그려 주고**, 목록으로 보여 줍니다.  
- 즉, **지도를 그리는 일의 대부분은 웹·서버 쪽**이 하고, 아두이노는 **“좌표와 시간을 보내는 일”**에 집중합니다.
- (최근 화면) 웹에는 **AI 기상 안전**, **본문 상단 AI 요약 자막**(한 바 스크롤 후 잠시 멈춤), **선박 신호(귀항·살포 시작/중지·위치 보고)**, **SOS 수신**, **항로·속도(시뮬·연동)** 도 함께 있습니다. 상세는 `docs/제품-운영/해양-종자-살포-관제-시스템-개요.md` 를 보시면 됩니다.

### 왜 지도 **API 키**를 아두이노에 넣지 않나요?

지도 API 키는 **“지도 그림(타일)을 받아오는 열쇠”**에 가깝습니다.  
그 열쇠를 배 위 기기에 박아 두면, 기기가 고장 나거나 잃어버릴 때 **남이 열쇠를 복사**해서 악용할 수 있습니다.  
그래서 **열쇠는 사무실(웹·서버) 쪽**에 두고, 배에서는 **“우리 집 주소(HTTPS 주소)로 편지만 보낸다”**고 생각하면 됩니다.

## 2. 역할 정리표 — “누가 뭘 하나요?”

| 일 | 아두이노(배 위) | 우리 웹 + 서버(육상) |
|----|-----------------|----------------------|
| GPS로 위치 잡기 | ✅ 한다 | ❌ 한다 않음 (브라우저 “내 위치”는 별개) |
| LTE로 인터넷 보내기 | ✅ 한다 | 받기만 함 |
| 살포 순간 좌표 보내기 | ✅ 버튼/센서에 맞춰 전송 | ✅ 받아서 DB에 저장 |
| 몇 분마다 배 궤적 보내기 | ✅ 정해진 주기로 전송 | ✅ 받아서 궤적으로 저장 |
| 지도에 점·선 그리기 | ❌ | ✅ 웹이 그림 |
| 로그인·권한·대시보드 | ❌ | ✅ 웹이 담당 |
| 너무 많은 요청 막기(남용 방지) | (기기는 적당히 보냄) | ✅ 서버에서 제한·검사 |

한 줄로 말하면:

> **아두이노 = “측정하고 보내는 쪽”**, **웹 = “모아서 보여주고 관리하는 쪽”** 입니다.

---

## 3. 코딩한 **다음에** 해야 할 일 (순서대로)

### ① 육상에서: “받을 우편함” 만들기 (한 번만)

- **데이터베이스**에 표(테이블)를 만들어 두고,  
- **서버 프로그램(Edge Function)** 이 “아두이노만 들어올 수 있는 문” 역할을 합니다.  
- 이건 보통 **IT 담당**이 SQL 실행·클라우드 설정으로 합니다.  
  (이 프로젝트에는 `scripts/sql/` 과 `supabase/functions/` 안에 예시가 있습니다.)

### ② 육상에서: “비밀 암호” 정하기

- 아무나 데이터 못 넣게 **비밀번호 같은 값**을 서버에만 저장합니다.  
- 아두이노 코드에는 그 값을 넣어서, 보낼 때마다 **같은 암호**를 함께 보냅니다.  
- 나중에 유출이 걱정되면 **암호만 바꾸면** 됩니다.

### ③ 배 위에서: 아두이노에 넣을 정보 적기

- **서버 주소**(HTTPS URL)  
- **암호**(위에서 정한 것)  
- **배 이름 문자열**(관제 화면과 똑같이, 예: 함정 이름)

### ④ 시험

- 먼저 **땅에서** WiFi/LTE로 한두 번 보내 보고,  
- 관제 웹에서 **점이나 선이 보이는지** 확인합니다.  
- 잘 되면 그다음 **실제 해상**에서 반복합니다.

### ⑤ 운영

- **몇 분마다** 궤적을 보낼지, **살포할 때만** 보낼지 운영 규칙을 정합니다.  
- 데이터 요금·배터리 때문에 “초당 여러 번”은 보통 필요 없습니다.

## 4. 자주 헷갈리는 것만 짚기

**Q. 아두이노가 지도를 띄우나요?**  
A. 꼭 그럴 필요 없습니다. 지도는 **관제실 PC 웹 브라우저**가 띄우고, 아두이노는 **숫자(위도·경도·시간)**만내면 됩니다.

**Q. 인터넷이 잠깐 끊기면?**  
A. 잘 만든 시스템은 기기 안에 **잠깐 저장해 두었다가**, 다시 연결되면 **순서대로 다시 보냅니다.** (큐)

**Q. 관공서 심사에서 뭐라고 말하면 되나요?**  
A. “현장 단말은 **인증된 수집 경로(HTTPS)**로만 데이터를 보내고, **지도·권한·저장**은 **중앙 서버·웹**에서 처리한다”고 말하면 됩니다.  
자세한 한 장 요약은 **`관공서-제출용/docs/관공서_설명용_해상종자살포_단말·연동요약.md`** 를 같이 두면 좋습니다.

**부품 선정·LTE·통신 범위·마커 주기·Arduino IDE·예제 스케치**는 저장소 **`arduino/`** 폴더를 열어보면 됩니다.

## 5. 그림으로만 기억하기

```
[ 배 위 ]                         [ 육상 / 인터넷 ]
  GPS ──► 좌표 알기
           │
  LTE  ────┼──► HTTPS ──► [ 서버가 받아서 저장 ]
           │                      │
           │                      └──► [ 관제 웹 지도·표 ]
  버튼/센서 ──► “지금 살포!” 같은 신호도 같이 보냄
```

아두이노는 **왼쪽**, 우리 웹은 **오른쪽 아래**까지 이어 주는 **전체 흐름의 절반**이라고 기억하면 됩니다.
