# 지식재산·정부 납품 전략 (프로젝트 내부 요약)

이 폴더는 **해상 종자살포 관제 시스템**을 정부·공공에 납품·판매하기 위한 연구, **나라장터 등 공고 알림**, **지식재산(특허·실용신안·상표·디자인) 및 긴급지원**을 한곳에서 다루기 위한 자료입니다.

## 원칙

- **원문 PDF·증빙**(신청서, 납부확인증, 사업자증명 등)은 사용자 PC에만 두고, 이 레포에는 **목차·체크리스트·절차**만 둡니다.
- 사업자번호, 연락처, 주소, 출원번호 등 **민감정보는 Git에 커밋하지 마세요.** 필요하면 아래 로컬 메모 파일만 사용합니다.

## 구성 파일

| 파일 | 용도 |
|------|------|
| `문서-인덱스.md` | 바탕화면 폴더에 있는 PDF 목록(로컬 경로) |
| `정부납품-크롤링-연구목표.md` | 납품·조달·OpenAPI 알림 연계 목표 |
| `공공정보수집-및-문의-루트.md` | 나라장터 외 부처·해양산업 정보원, 관공서 문의·이메일 루트 |
| `지식재산긴급지원-체크리스트.md` | 긴급지원·출원 연계 일반 체크리스트 |
| `개인정보-로컬메모.md.example` | 복사 후 `개인정보-로컬메모.md`로 저장해 로컬에만 기입 |

## 코드와의 연결

- 나라장터 **용역·물품·공사** 공고 조회·알림: `scripts/g2b-bid-watch.mjs`, `pnpm run g2b:watch` (`G2B_BID_TYPES`)
- **관공서 제출용 화면 캡처**: `관공서-제출용/` + `pnpm run capture:gov` (`pnpm dev` 선행)
- 환경 변수 예시: `.env.example`의 `DATA_GO_KR_SERVICE_KEY`, `G2B_KEYWORDS` 등

## 에이전트 동작

`.cursor/rules/marine-gov-ip-strategy.mdc` 에서 이 폴더를 우선 참고하도록 했습니다. 조달·지재권·납품 질문 시 **추가할까요를 반복해서 묻지 않고**, 관련 항목(용역·물품 API 확장, 보안·계약, 실적 요건 등)은 **바로 제안·반영**하는 것을 원칙으로 합니다.
