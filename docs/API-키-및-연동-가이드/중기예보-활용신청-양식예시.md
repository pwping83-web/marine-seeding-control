# 기상청_중기예보 조회서비스 — 활용신청 화면 채우기 예시

공공데이터포털 **「JSON+XML 기상청_중기예보 조회서비스」** 활용신청 시 붙여 넣거나 참고할 수 있는 예시입니다.  
법적 판단·반려 여부는 **포털·기상청 안내가 우선**이며, 아래는 일반적인 내부·시연용 웹 관제 화면 기준입니다.

---

## 1. 활용목적 (체크박스)

| 권장 | 설명 |
|------|------|
| **웹 사이트 개발** | 본 레포는 React(Vite) 웹 관제 화면에서 API를 호출합니다. 해당 항목에 체크하는 것이 자연스럽습니다. |
| 앱 개발 | 모바일 네이티브 앱을 따로 만들지 않으면 생략 가능. |
| 연구(논문 등) | 학술 목적이면 추가 체크. 일반 운영·납품용이면 생략해도 됩니다. |

---

## 2. 활용목적 (자유 입력, 250자 이내) — 복사용 예시

아래 중 하나를 그대로 쓰거나, 기관명·함명만 바꿔 사용하세요.

**예시 A (짧음, 약 120자)**

```
해양 종자 살포 관제용 웹 애플리케이션에서 작업 계획(일별 달력)에 기상청 중기예보(육상·해상·기온)를 반영하여 출항·살포 가능 여부를 참고 표시합니다. 내부 시연·실증 목적이며 상업적 위치정보 서비스 제공이 아닙니다.
```

**예시 B (조금 상세, 약 200자)**

```
Vite 기반 관제 웹에서 MidFcstInfoService(중기육상·중기해상·중기기온)를 호출해 예보일 3~10일 구간의 날씨·파고·기온을 작업 일정 UI에 표시합니다. 선박 운항 안전 보조·내부 보고용이며, 제3자에게 위치기반 유료 서비스를 제공하지 않습니다.
```

## 3. 상세기능정보 선택

본 저장소 `src/lib/kma-weather.ts`의 `fetchKmaMiddleTermForecast`는 아래 **3개 연산**을 사용합니다.

| 상세기능 | 코드에서 호출 | 선택 |
|----------|----------------|------|
| **중기육상예보조회** | `getMidLandFcst` | ✅ 필수 |
| **중기기온조회** | `getMidTa` | ✅ 필수 |
| **중기해상예보조회** | `getMidSeaFcst` | ✅ 필수 |
| **중기전망조회** | (현재 미사용) | 선택 — 나중에 전망 문구만 따로 쓸 계획이 있으면 체크 |

**정리:** 최소한 **육상·기온·해상** 세 줄은 체크하세요. 전망은 없어도 동작합니다.

## 4. 첨부파일 · 위치기반서비스 안내 문구

포털에 다음과 같은 안내가 있습니다.

> 위치정보를 포함한 서비스를 사용하고자 하는 사업자는 … 방송통신위원회에 **위치정보서비스 허가** 또는 **위치기반 서비스사업 신고** … 첨부파일에 **위치기반서비스사업신고필증**을 첨부

**일반적인 이해(참고용, 법률 자문 아님):**

- 본 관제 웹은 **기상 예보 구역 코드(regId)** 로 API를 조회하고, **지도는 OSM 타일 + 선박·살포 좌표 표시** 수준인 경우가 많습니다.
- **「다른 사람의 위치를 상품으로 제공하는 위치기반 사업」**과 같지 않다면, 신고 대상인지 여부는 **사업 형태·기관 내부 규정**에 따라 달라질 수 있습니다.
- **불확실하면:** 방송통신위원회·소속 기관 정보화 담당에 문의하거나, 신고필증이 있으면 첨부하는 편이 안전합니다.
- **첨부가 없을 때 반려**된다는 문구가 있으므로, 반려 시 포털 1:1 문의 또는 기상청 API 담당 안내를 활용하세요.

## 6. 승인 후 `.env` 설정

단기예보와 **동일한** `VITE_KMA_SERVICE_KEY`를 쓸 수 있습니다(키 문자열 하나로 여러 데이터셋이 허용되는 경우가 많음).  
별도로 중기만의 키가 발급되면 그 값을 넣으면 됩니다.

```env
VITE_KMA_SERVICE_KEY=발급받은_인증키
# 선택 — 해역에 맞게 API 문서의 regId로 변경
# VITE_KMA_MIDLAND_REGION=11H20000
# VITE_KMA_MIDTA_REGION=11H20000
# VITE_KMA_MIDSEA_REGION=12B20000
```

자세한 변수 설명은 같은 폴더의 `README.md`를 참고하세요.
