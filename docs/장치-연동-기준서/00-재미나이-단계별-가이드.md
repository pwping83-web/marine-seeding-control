# 00. 재미나이(Gemini)용 — 단계별로 물어보는 방법

> **이 문서만 먼저 읽으면 됩니다.**  
> 장치 연동 기준서 폴더 전체가 길어서, **한 번에 하나의 파트**만 재미나이에게 맡기세요.  
> 제품을 아직 모를 때는 **파트 1 → 2**에서 “무엇을 사야 하는지”만 정리한 뒤, 사고 나서 **파트 3 이후**로 가면 됩니다.

---

## 재미나이에 올릴 때 (공통)

1. **폴더**: `docs/장치-연동-기준서/` 전체를 ZIP으로 압축해 업로드하거나, 해당 폴더를 드래그해 넣습니다.
2. **첫 메시지 예시** (복사해서 쓰기):

   ```text
   첨부한 폴더는 "해양 종자 살포 관제" 장치 연동 기준서입니다.
   나는 지금 [파트 번호] 단계입니다. 아래 "해야 할 일"을 순서대로 도와줘.
   한 번에 한 단계만 설명하고, 다음 단계는 내가 "다음"이라고 할 때만 이어줘.
   ```

3. **규칙**: 재미나이가 한꺼번에 길게 쓰면, **「지금은 파트 N만」**이라고 다시 요청하세요.

---

## 파트 0 — 이 가이드와 목록만 맞추기

| 해야 할 일 | 완료 |
|------------|------|
| 이 파일(`00-재미나이-단계별-가이드.md`)을 연다 | ☐ |
| `README.md`의 문서 목록(01~06)이 무엇인지 훑어본다 | ☐ |

**재미나이에게 물어볼 말 (예시)**

```text
00번과 README만 기준으로, 01~06 각 파일이 언제 필요한지 한 줄씩만 정리해줘.
```

## 파트 1 — 전체 그림만 이해하기 (부품 사기 전)

**목표**: “배 위 장치 → 인터넷 → 서버 → 관제 웹 지도” 흐름만 이해한다.

| 읽을 문서(우선순위) | 역할 |
|---------------------|------|
| `README.md` | 한눈에 보기 |
| `03-선박-지도-표시-구조.md` | 지도에 배가 잡히는 흐름 |
| (선택) `07-아두이노-GNSS-살포-연동-실무-가이드.md` | 웹「테스트 살포」와 DB 한 줄이 어떻게 같은지 |

| 해야 할 일 | 완료 |
|------------|------|
| 위 두 문서를 재미나이와 함께 훑는다 | ☐ |
| `vessel_id` / `vessel_track_points` / `vessel_positions` 차이를 말로 설명할 수 있게 한다 | ☐ |

**재미나이에게 물어볼 말 (예시)**

```text
파트 1이야. README와 03번만 보고, 비전문가에게 설명하듯 "신호가 어디로 가는지" 순서만 5줄 이내로 알려줘.
```

## 파트 2 — 무엇을 살지 정하기 (제품 모를 때 여기서 멈춰도 됨)

**목표**: MCU, GPS, 통신(LTE 또는 WiFi), 투하 신호 받는 방법까지 **후보 1~2개**로 좁힌다.

| 읽을 문서 | 역할 |
|-----------|------|
| `01-기기사양-기준서.md` | 최소 사양, 권장 모듈, 링크 |
| `06-실제-연동-방법과-문제점.md` (앞부분) | 현장에서 자주 막히는 것 |

| 해야 할 일 | 완료 |
|------------|------|
| 한국에서 쓸 통신 방식 정하기: **항만 WiFi만** vs **LTE 유심** | ☐ |
| GPS 모듈 후보 정하기 (예: NMEA UART 나오는 모듈) | ☐ |
| 투하 신호: **버튼/건접점** vs **PLC** 중 어디에 맞출지 정하기 | ☐ |
| 쇼핑몰 링크로 장바구니에 넣기 전, 재미나이에게 “이 조합 되나?” 확인 | ☐ |

**재미나이에게 물어볼 말 (예시)**

```text
파트 2야. 01번 기기사양만 보고, 나는 [WiFi만 / LTE 유심]으로 하려고 해.
MCU+GPS+통신 조합으로 살 목록을 표로 최소한만 적어줘. 한국 LTE면 밴드 확인할 것도 같이 적어줘.
```

## 파트 3 — 서버(Supabase) 쪽 손대기

**목표**: Edge Function·DB·비밀키까지 **한 번** 맞춰 둔다.

| 읽을 문서 | 역할 |
|-----------|------|
| `06-실제-연동-방법과-문제점.md` **1. 연동 전 준비** | Supabase, Secret, 배포 명령 |
| `02-신호-프로토콜-명세.md` | 헤더·JSON 형식 |

| 해야 할 일 | 완료 |
|------------|------|
| `DEVICE_INGEST_SECRET` 생성·등록 | ☐ |
| `vessel-track-ingest`, `telemetry-ingest` 배포 | ☐ |
| 관제 웹 `.env`에 `VITE_VESSEL_LTE_ID` 등 넣기 | ☐ |
| PC에서 `curl`로 02번 예시대로 HTTP 200 확인 | ☐ |

**재미나이에게 물어볼 말 (예시)**

```text
파트 3이야. 06번 문서의 「1. 연동 전 준비」와 02번을 보고, 내가 Supabase에서 순서대로 해야 할 체크리스트만 적어줘. curl 예시는 OS가 [Windows / Mac] 기준으로 하나만.
```

## 파트 4 — 코드 업로드·수정 (Arduino / ESP32)

**목표**: `readGpsFix()` 실제 UART, `VESSEL_ID`·URL·Secret, 전송 주기를 본인 환경에 맞춘다.

| 읽을 문서 | 역할 |
|-----------|------|
| `02-신호-프로토콜-명세.md` | JSON 필드 그대로 맞추기 |
| `06-실제-연동-방법과-문제점.md` **2. ESP32 스케치 설정** | 스케치에서 바꿀 5가지 |
| `04-투하-작동-구조.md` | GPIO13, 디바운스, 안전 로직 |

| 해야 할 일 | 완료 |
|------------|------|
| 스케치 상단 5개 상수( WiFi / REF / Secret / VESSEL_ID / USE_ESP32_HTTP ) 수정 | ☐ |
| GPS 실제 파싱으로 교체 | ☐ |
| LTE 보드면 TinyGSM 쪽으로 HTTP 경로 맞추기 (06 참고) | ☐ |
| USB로 시리얼에 JSON이 찍히는지 확인 후, `USE_ESP32_HTTP 1` | ☐ |

**재미나이에게 물어볼 말 (예시)**

```text
파트 4야. 내 스케치 코드를 붙여넣을 테니(또는 이 저장소의 esp32 경로 알려줄게), 02번 JSON 형식이랑 맞는지 필드만 표로 짚어줘. 고치라고 할 때만 코드 제안해줘.
```

## 파트 5 — 배선·전원·투하 테스트 (책상 위)

**목표**: 화면 없이도 **시리얼·로그**로 “투하 한 번 = 기록 한 번”이 되는지 본다.

| 읽을 문서 | 역할 |
|-----------|------|
| `01-기기사양-기준서.md` **4. 투하장치 연동** | 투하 신호 방식(건접점 등) |
| `04-투하-작동-구조.md` | DROP_TRIGGER_ON_HIGH 의미 |
| `06-실제-연동-방법과-문제점.md` **3. 현장 배선 절차** | 배선 순서 |

| 해야 할 일 | 완료 |
|------------|------|
| ESP32만 USB로 부팅 로그 확인 | ☐ |
| GPS 연결 후 `$GPRMC` 또는 좌표가 시리얼에 나오는지 확인 | ☐ |
| GPIO13을 GND에 잠깐 닿혀 살포 이벤트 1건 시뮬레이션 | ☐ |
| Supabase `seed_drop_records`에 행이 생기는지 확인 | ☐ |

**재미나이에게 물어볼 말 (예시)**

```text
파트 5야. 04번 기준으로 내 투하 장치는 [평소 HIGH / 평소 LOW]인데, DROP_TRIGGER_ON_HIGH는 true/false 중 뭐가 맞아? 한 문장으로만.
```

## 파트 6 — 관제 웹과 같이 보는 통합 테스트

**목표**: 지도에 **항적 선**·**살포 마커**가 의도대로 보이는지 확인한다.

| 읽을 문서 | 역할 |
|-----------|------|
| `03-선박-지도-표시-구조.md` | 마커 vs 항적 차이 |
| `05-항해경로-시작-종료.md` | 항해 시작/종료는 웹에서 |

| 해야 할 일 | 완료 |
|------------|------|
| `VESSEL_ID`와 `VITE_VESSEL_LTE_ID` 문자열이 **완전히 동일**한지 확인 (공백 없음) | ☐ |
| `vessel_track_points`에 점이 쌓이는지 확인 | ☐ |
| 살포 마커가 뜨는지 확인 | ☐ |
| “선박 아이콘은 안 움직이고 선만 간다”면 03·06의 `vessel_positions` 설명 읽기 | ☐ |

**재미나이에게 물어볼 말 (예시)**

```text
파트 6이야. 03번만 보고, "주황 항적은 보이는데 배 아이콘은 안 움직인다"면 원인 한 가지와 해결 옵션 두 가지만 bullet로.
```

## 나(사용자)가 Cursor/코딩 에이전트에게 맡길 때

재미나이는 글·순서 정리에 좋고, **실제 저장소 코드 수정·빌드**는 이 프로젝트의 Cursor 대화에 맡기면 됩니다.

| 상황 | Cursor에게 할 말 예시 |
|------|-------------------------|
| 스케치 수정 | `esp32_marine_telemetry.ino`에서 readGpsFix와 vessel JSON을 02번에 맞춰줘 |
| 웹 마커 동기화 | LTE 최신 점으로 vessel_positions upsert 하는 패치 해줘 |
| 문서만 바꿀 때 | 장치 연동 기준서 03번에 ○○ 문단 추가해줘 |
