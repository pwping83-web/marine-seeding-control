# 04. 투하 작동 구조 — 어떤 작동에 의해 투하가 진행되는가

---

## 1. 투하 트리거 방식 개요

투하(살포) 이벤트는 두 가지 방식으로 발생합니다.

| 방식 | 트리거 | 적합한 경우 |
|------|--------|------------|
| **A. 수동 버튼** | 작업자가 GPIO13에 연결된 스위치를 누름 | 수동 살포, 단순 현장 |
| **B. 자동 PLC 신호** | 투하장치 PLC가 GPIO13에 전기 펄스를 출력 | 자동 투하 시스템, 컨베이어 연동 |

---

## 2. 전체 투하 흐름 다이어그램

```
[투하장치 / PLC / 수동 버튼]
          │
          │ 전기 신호 (펄스 또는 건접점 단락)
          ▼
[ESP32 GPIO 13 — INPUT_PULLUP]
          │
          │ tickSeedDrop() 매 루프 폴링
          ▼
[에지 검출 + 안전 로직]
    ├── 부팅 유예 3초 이내? → 무시
    ├── 이전 살포 후 800ms 이내? → 무시
    ├── 60초 내 24회 초과? → 무시
    ├── GPS 좌표 유효? → 범위 이탈 시 무시
    └── 통과 → sendDropTelemetryNow() 호출
          │
          │ JSON 빌드
          ▼
[HTTPS POST → telemetry-ingest]
          │
          │ 서버 인증·검증·레이트 리밋
          ▼
[seed_drop_records 테이블 UPSERT]
          │
          │ 관제 웹 폴링(12초)
          ▼
[지도에 살포 원형 마커 표시]
[살포 이력 목록에 1행 추가]
```

## 3. 투하 신호 GPIO 상세 동작

### 3-1. INPUT_PULLUP 핀 상태

```
평소 상태 (연결 없음):
  GPIO13 = HIGH (3.3V, 내부 풀업 저항으로 유지)

투하 장치 동작 시 (건접점 방식):
  GPIO13 → GND (LOW)
  
투하 장치 동작 시 (NPN 트랜지스터/오픈드레인 방식):
  트랜지스터 포화 → GPIO13 → GND (LOW)
```

### 3-2. DROP_TRIGGER_ON_HIGH 설정에 따른 에지 판정

| 설정값 | 트리거 에지 | 사용 경우 |
|--------|------------|-----------|
| **`true` (스케치 기본값)** | **상승 에지** (LOW → HIGH) | PLC 양극 펄스, 24V OFF→ON 신호, 평소 LOW 유지 장치 |
| `false` | **하강 에지** (HIGH → LOW) | 일반 택트 스위치(풀업), 건접점 단락(NO→GND), NPN 출력 |

> **배선 결정 기준**: 투하장치 신호선이 **평소 LOW이고 투하 시 HIGH**가 되면 `true`, **평소 HIGH(풀업)이고 투하 시 LOW(GND 단락)**이면 `false`.

```cpp
// 에지 판정 로직
const bool edgeUp   = (prevStable == LOW  && stabilized == HIGH);
const bool edgeDown = (prevStable == HIGH && stabilized == LOW);
const bool fire = (DROP_TRIGGER_ON_HIGH && edgeUp) 
               || (!DROP_TRIGGER_ON_HIGH && edgeDown);
```

### 3-3. 채터링 방지 (디바운스)

```
신호 변화 감지 후 DROP_DEBOUNCE_MS(50ms) 동안 안정 상태 유지 확인:

[DROP_TRIGGER_ON_HIGH=true, 상승 에지 방식 예시]

GPIO13: LOW  ─────────┐  ┌─ 50ms ─┐  ┌────────
                      └──┘ (채터) └──┘
             HIGH          HIGH 안정화↑
                                이 시점에 상승 에지 처리

[DROP_TRIGGER_ON_HIGH=false, 하강 에지 방식 예시]

GPIO13: HIGH ─────────┐  ┌─ 50ms ─┐  ┌────────
                      └──┘ (채터) └──┘
             LOW           LOW 안정화↑
                                이 시점에 하강 에지 처리

이중 샘플링 (digitalReadStable):
  const int a = digitalRead(pin);
  delayMicroseconds(30);  // 30μs 후
  const int b = digitalRead(pin);
  return (a == b) ? a : a;  // 두 번 읽어 단발 노이즈 완화
```

## 4. 안전 로직 5단계

투하 오발 방지를 위해 아래 5가지를 **모두** 통과해야 투하 기록이 전송됩니다. (코드 내에서 병렬·중첩 체크되며 순서가 고정되지 않는 항목도 있음)

```
① 부팅 유예 (SEED_DROP_BOOT_GRACE_MS = 3000ms)
   ─ 전원 ON 후 3초 동안 어떤 에지도 무시
   ─ 이유: 전원 투입 시 GPIO 리플, 부팅 진동으로 인한 오발 방지

② 채터링 방지 (DROP_DEBOUNCE_MS = 50ms)
   ─ 신호 변화 후 50ms 이내 재변화는 채터로 간주

③ 이중 샘플링 (digitalReadStable)
   ─ 30μs 간격으로 두 번 읽어 일치하지 않으면 1차 값 사용

④ 최소 간격 (SEED_DROP_MIN_INTERVAL_MS = 800ms)
   ─ 직전 투하 시도 후 800ms 이내 다시 발생하면 무시
   ─ 이유: 기계 진동이나 빠른 이중 접촉으로 인한 연속 전송 방지

⑤ 60초 시도 한도 (SEED_DROP_MAX_ATTEMPTS_PER_60S = 24회)
   ─ 슬라이딩 60초 창 안에 24회 초과 시 이후 모두 무시
   ─ 이유: 센서 오동작·핀 단락 등 폭주 상황에서 서버 부하 방지
```

## 5. 모터·릴레이 연동 (자동 투하 시스템)

`esp32-marine-telemetry-motor-relay` 스케치를 사용할 때 릴레이로 투하 모터/밸브를 직접 제어합니다.

### 5-1. 시작/정지 흐름

```
GPIO10 (시작/정지 버튼) 눌림 (HIGH→LOW 하강 에지)
         │
         ├── [현재 running = false]
         │     GPIO9 비상정지 활성? → "비상정지 중" 로그
         │     부팅 유예·쿨다운 이내? → "잠시 후 시작" 로그
         │     통과 → running = true
         │            phase = 0 (A 릴레이 시작)
         │            GPIO11(릴레이 A) = ON
         │            GPIO12(릴레이 B) = OFF
         │
         └── [현재 running = true]
               running = false
               GPIO11 = OFF, GPIO12 = OFF
               시작 쿨다운 타이머 리셋
```

### 5-2. 릴레이 교대 타이밍

```
시작 후:
  Phase 0: GPIO11(릴레이 A) ON ─── RELAY_TIME_MS_PIN_11(2000ms) ───▶ Phase 1
  Phase 1: GPIO12(릴레이 B) ON ─── RELAY_TIME_MS_PIN_12(2000ms) ───▶ Phase 0
  
  이 교대를 정지 명령 또는 비상정지까지 무한 반복
  
타임라인 예시 (RELAY_TIME = 2000ms):
  0ms:    릴레이 A ON  → 밸브 A 열림 → 종자 투하
  2000ms: 릴레이 A OFF, 릴레이 B ON → 밸브 B 열림
  4000ms: 릴레이 B OFF, 릴레이 A ON → 반복
```

### 5-3. 비상정지

```
GPIO9 (비상정지 버튼) → LOW 감지 (EMERGENCY_STOP_ACTIVE_LOW = true)
         │
         ▼
  running = false
  GPIO11 = OFF (릴레이 A 즉시 OFF)
  GPIO12 = OFF (릴레이 B 즉시 OFF)
  시작 쿨다운 시작 (MOTOR_START_COOLDOWN_MS)
  
  ※ 비상정지 해제(GPIO9 = HIGH) 전까지 GPIO10 시작 불가
```

## 6. 투하 신호가 서버에 기록되는 전체 타임라인

```
T+0:000ms  투하 장치 동작 (PLC 출력 ON 또는 버튼 누름)
T+0:050ms  GPIO13 안정화 확인 (DROP_DEBOUNCE_MS 경과)
T+0:050ms  에지 판정 통과 → sendDropTelemetryNow("A01") 호출
T+0:050ms  GPS 현재 위치 읽기 (readGpsFix)
T+0:050ms  JSON 직렬화:
           {"records":[{
             "id":"ship-제3해양살포함-1747177200050",
             "label":"A01",
             "drop_time":"14:30:00",
             "lat":34.8215,"lng":128.5235,
             "status":"성공",
             "recorded_at":1747177200050,
             "verification_mismatch":false
           }]}
T+0:100ms  HTTPS POST 시작 (TLS 핸드셰이크 시작)
T+0:500ms~ HTTPS POST 완료 (LTE 지연에 따라 500ms~3초)
           → telemetry-ingest: HTTP 200 {"ok":true,"accepted":1}
           → seed_drop_records에 UPSERT 완료
T+12초     관제 웹 다음 폴링 주기
           → seed_drop_records SELECT
           → 지도에 살포 마커 표시
           → 살포 이력 목록에 1행 추가
```

## 7. 관제 웹 살포 표시 요소

| 화면 요소 | 표시 조건 | 데이터 원천 |
|-----------|-----------|-------------|
| 지도 원형 마커 | 유효한 lat/lng 있는 seed_drop_records | `seed_drop_records.lat`, `.lng` |
| 마커 색상 | status 값에 따라 다름 | `seed_drop_records.status` |
| 마커 라벨 | 라벨 문자열 표시 | `seed_drop_records.label` |
| 이력 목록 | 투하 시각·위치·상태 | `seed_drop_records` 전체 |
| 살포 통계 | 오늘 날짜 기준 건수 집계 | `seed_drop_records.drop_time` |

## 8. 투하 기록 미수신 시 원인 점검

```
[ESP32 시리얼 모니터 확인]
  "[seed-drop] 안전: 간격 또는 60초 시도 한도" 
  → 너무 빠르게 연속 신호 발생. 장치 배선 점검.
  
  "[seed-drop] 안전: 위·경도 범위 이상"
  → GPS 미수신 상태에서 투하. GPS 안테나 위치 점검.
  
  "[telemetry-ingest] HTTP fail"
  → LTE 연결 끊김. 신호 강도(RSSI) 점검.

[서버 로그 확인 (Supabase Dashboard → Edge Functions → telemetry-ingest)]
  "invalid_device_key" → DEVICE_INGEST_SECRET 불일치
  "invalid_record_shape" → status 값이 "성공"/"실패"/"대기" 외의 값
  "rate_limited_per_ip" → 너무 짧은 주기로 대량 전송

[DB 직접 확인]
  SELECT * FROM seed_drop_records ORDER BY recorded_at DESC LIMIT 10;
  → 최근 기록 있으면 서버 도달은 성공. 관제 웹 폴링 문제.
```
