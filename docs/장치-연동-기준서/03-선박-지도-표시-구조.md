# 03. 선박이 지도에 표시되는 전체 신호 흐름

> "배에 연결된 장치가 어떤 구조로 신호를 보내어 지도에 배가 표시되는가"

## 1. 전체 흐름 다이어그램

```
┌─────────────────────────────────────────────────────────────────────┐
│                          [배 위 — 하드웨어]                          │
│                                                                      │
│  위성 (GPS/GNSS)                                                     │
│       │ NMEA 신호 (위성 전파)                                        │
│       ▼                                                              │
│  GPS 모듈 (NEO-M8N 등)                                              │
│       │ UART (9600 baud, $GPRMC/$GPGGA 문장)                        │
│       ▼                                                              │
│  ESP32 MCU                                                           │
│    readGpsFix() → lat, lng, speed, heading                          │
│    tickVesselTrack() : POST_INTERVAL_MS(60초)마다                   │
│       │ JSON 직렬화                                                  │
│       │ {"vessel_id":"제3해양살포함","points":[{...}]}              │
│       │                                                              │
│  LTE 모듈 (SIM7600 등)                                              │
│       │ HTTPS POST (LTE 셀룰러 데이터)                               │
│       │ Content-Type: application/json                               │
│       │ X-Device-Ingest-Key: SECRET                                 │
└───────┼─────────────────────────────────────────────────────────────┘
        │ 인터넷 (LTE → 기지국 → 백본망)
        ▼
┌─────────────────────────────────────────────────────────────────────┐
│                       [Supabase Cloud — 서버]                        │
│                                                                      │
│  Edge Function: vessel-track-ingest                                  │
│    ① X-Device-Ingest-Key 인증 확인                                  │
│    ② JSON 유효성 검사 (좌표 범위, vessel_id 형식)                   │
│    ③ 레이트 리밋 확인 (IP당 15건/초)                                │
│    ④ vessel_track_points 테이블에 INSERT                            │
│                                                                      │
│  PostgreSQL DB                                                        │
│    vessel_track_points (행 누적: 시간·좌표·속도·방위)               │
│    vessel_positions    (1척 1행, 현재 위치만 upsert)                 │
│                                                                      │
│  Supabase Realtime (WebSocket)                                       │
│    vessel_positions 변경 이벤트 → 구독 클라이언트에 푸시            │
└───────┼─────────────────────────────────────────────────────────────┘
        │ HTTPS (Supabase JS 클라이언트) + WebSocket (Realtime)
        ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     [관제 웹 — 브라우저]                             │
│                                                                      │
│  vessel-position-db.ts                                               │
│    subscribeVesselPositions() → Realtime 구독                       │
│    vessel_positions 변경 → onUpdate() 콜백                          │
│                                                                      │
│  MarineLeafletMap.tsx                                                │
│    vessel_positions 데이터 → Leaflet 마커 갱신                      │
│    vessel_track_points 폴링 (VITE_VESSEL_LTE_POLL_MS, 기본 12초)   │
│    → 주황색 실선(Polyline) 그리기                                   │
│                                                                      │
│  지도 화면                                                           │
│    ▶ 선박 아이콘 마커 (현재 위치)                                   │
│    ▶ 주황색 항적 선 (지나온 경로)                                   │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 단계별 상세 설명

### 단계 1: GPS 위성 수신 (배 위)

```
위성 (GPS/GLONASS/Beidou) → GPS 모듈 안테나 → GPS 모듈 내부 수신기
```

- GPS 모듈은 위성 신호를 수신하여 **NMEA-0183 텍스트 문장**을 UART로 출력
- 1초마다(또는 설정된 주기마다) `$GPRMC`, `$GPGGA` 등의 문장 출력
- ESP32 UART RX 핀이 이 문장을 수신 → [TinyGPS++](https://github.com/mikalhart/TinyGPSPlus) 또는 수동 파싱으로 lat/lng 추출

**주의사항**:
- 위성 첫 수신(`Cold Start`)에 최대 30초~3분 소요
- 선박 상부 금속 구조물이 안테나를 가리면 수신 불가
- 안테나는 **하늘이 보이는 최상부**에 설치

### 단계 2: ESP32 데이터 처리

```cpp
// tickVesselTrack() 함수 — 매 루프마다 호출, 주기 확인
void tickVesselTrack(unsigned long now) {
    // 60초 경과 확인
    if (now - lastVesselPostMs < POST_INTERVAL_MS) return;
    
    // GPS 데이터 읽기
    double lat, lng;
    float speed, heading;
    if (!readGpsFix(lat, lng, speed, heading)) return;  // 수신 실패 시 건너뜀
    
    // 좌표 유효성 검사 (0,0 또는 범위 이탈 차단)
    if (!geoPlausibleDeg(lat, lng)) return;
    
    // JSON 빌드 → HTTPS POST
    // ...
    lastVesselPostMs = now;  // 다음 주기까지 타이머 리셋
}
```

### 단계 3: LTE HTTPS 전송

- WiFi 또는 LTE 모듈이 인터넷에 연결된 상태에서 Supabase Edge Function URL로 HTTPS POST
- SSL/TLS 암호화 (WiFiClientSecure, 현장 운영 시 CA 검증 권장)
- 타임아웃: 20초 (해상 LTE 지연 고려)

### 단계 4: 서버(Edge Function) 처리

```
vessel-track-ingest 처리 순서:
  1. OPTIONS 프리플라이트 → CORS 허용
  2. X-Device-Ingest-Key 헤더 확인 → 불일치: 401
  3. Content-Length ≤ 64KB 확인 → 초과: 413
  4. JSON 파싱 → vessel_id, points[] 추출
  5. 각 point의 lat/lng 범위, recorded_at 범위 검사
  6. api_rate_try RPC로 레이트 리밋 확인
  7. vessel_track_points 테이블 INSERT
  8. HTTP 200 {"ok":true,"accepted":N} 반환
```

### 단계 5: DB 저장 → 관제 웹 반영

**A. vessel_track_points (항적 누적)**

```sql
-- ESP32가 60초마다 INSERT하는 행
INSERT INTO vessel_track_points 
  (vessel_id, recorded_at, lat, lng, speed_kn, heading_deg, source)
VALUES
  ('제3해양살포함', '2026-05-14T14:30:00Z', 34.82, 128.52, 4.2, 45.0, 'arduino-lte');
```

관제 웹은 `VITE_VESSEL_LTE_POLL_MS`(기본 12초)마다 이 테이블을 SELECT하여 지도에 주황 선으로 표시합니다.

**B. vessel_positions (현재 위치)**

`vessel_positions`는 **`vessel-track-ingest` Edge Function이 갱신하지 않습니다.** 이 테이블은 아래 두 경로로만 업데이트됩니다.

| 경로 | 설명 |
|------|------|
| **브라우저 GPS (스마트폰·PC)** | 관제 웹에서 선박 담당 계정(`VESSEL_USER_EMAIL`)으로 로그인 시 `navigator.geolocation`으로 브라우저 GPS를 읽어 `upsertVesselPosition()` 호출 |
| **수동 upsert** | Supabase SQL Editor 또는 서버 측 코드에서 직접 삽입·갱신 |

> **실무 중요**: ESP32 LTE 단말이 `vessel-track-ingest`로 전송한 데이터는 `vessel_track_points`(항적 이력)에만 저장됩니다. **현재 위치 마커(`vessel_positions`)는 별도로 갱신하지 않으면 움직이지 않습니다.** 실시간 마커가 필요하면 관제 웹에서 `vessel_track_points`의 가장 최신 점을 읽어 마커를 이동시키는 추가 로직이 필요합니다.

- Supabase Realtime이 `vessel_positions` 변경을 WebSocket으로 즉시 브라우저에 푸시
- 브라우저는 [Leaflet](https://leafletjs.com/) 마커 위치를 바로 갱신

## 5. `vessel_id` 불일치 문제

지도에 선박이 안 보이는 가장 흔한 원인입니다.

| ESP32 스케치의 `VESSEL_ID` | 관제 웹 `.env`의 `VITE_VESSEL_LTE_ID` | 결과 |
|---------------------------|--------------------------------------|------|
| `"제3해양살포함"` | `"제3해양살포함"` | ✅ 지도에 표시됨 |
| `"제3해양살포함"` | `"vessel-01"` | ❌ 지도에 안 보임 |
| `"제3해양살포함 "` (공백) | `"제3해양살포함"` | ❌ 안 보임 (공백 주의!) |

> **체크**: `vessel_track_points` 테이블에서 `SELECT DISTINCT vessel_id`로 실제 저장된 ID 확인 후 웹 환경변수와 맞추세요.

## 6. 실시간 현재 위치 vs 항적 폴링 구조 차이

```
[실시간 현재 위치 — vessel_positions]
브라우저(선박 로그인 계정) navigator.geolocation
  → Supabase JS SDK upsertVesselPosition()
  → vessel_positions 테이블 (vessel_id PK, upsert)
  → Supabase Realtime (PostgreSQL WAL → WebSocket)
  → 브라우저 subscribeVesselPositions() onUpdate() 콜백
  → [Leaflet](https://leafletjs.com/) 마커 즉시 이동

[항적 이력 폴링 — vessel_track_points]
ESP32 → HTTPS POST → vessel-track-ingest → vessel_track_points INSERT (행 누적)
  → 브라우저 setInterval(폴링, VITE_VESSEL_LTE_POLL_MS)
  → SELECT FROM vessel_track_points WHERE vessel_id='...' ORDER BY recorded_at
  → [Leaflet](https://leafletjs.com/) Polyline 주황 선 업데이트
```

> **두 경로는 독립적입니다.** `vessel_positions` 갱신 없이 ESP32만 동작하면 지도의 실시간 **마커는 움직이지 않고** 주황 항적 선만 늘어납니다. 실시간 마커까지 필요하면 관제 웹에서 `vessel_track_points` 최신 점으로 `vessel_positions`를 자동 동기화하는 로직을 추가해야 합니다.
