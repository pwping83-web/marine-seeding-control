# 해양 종자 살포 관제 시스템 — 개요 및 작동 방식

## 문서 목적

본 문서는 현재 저장소에 구현된 **해양 종자 살포 관제** 웹 화면의 **컨셉**, **동작 흐름**, **기술 구성**, **백엔드 유무**를 한글로 정리합니다. 실제 관공서·내부망 연동 전 **시연·납품용 UI 프로토타입** 기준입니다.

---

## 시스템 컨셉

- **대상**: 해조류 복원 등을 위한 **종자 살포** 작업을 **실시간에 가깝게** 모니터링하는 관제 데스크톱 UI.
- **톤**: 어두운 남색 계열, 통계·이력·지도가 한 화면에 모이는 **행정·관제용 대시보드** 스타일.
- **지도**: 연안 실루엫·항적·살포 점을 **SVG 기반 플레이스홀더**로 표현. 화면 하단 문구에 따라 **카카오맵·구글맵 등 외부 지도 API 연동은 예정**으로 표기되어 있습니다.
- **데이터**: 살포 이력·선박 위치·일부 통계는 **브라우저 내부 시연용 목(mock)** 이며, 별도 DB·센서와 연결되어 있지 않습니다.

---

## 백엔드 서버 및 API 현황

| 항목 | 현재 상태 |
|------|-----------|
| **백엔드 서버** | **없음.** Node/Express, Spring, FastAPI 등 별도 API 서버는 포함되어 있지 않습니다. |
| **HTTP API 호출** | 로그인·이력 조회·살포 기록 전송 등 **외부 REST/GraphQL 호출 없음.** |
| **인증 서버** | 없음. 로그인은 **클라이언트만** 처리합니다. |
| **데이터 저장** | 페이지 새로고침 시 **초기 시드 데이터로 리셋**됩니다(영구 저장소 없음). |

향후 실서비스에 맞추려면 예를 들어 다음이 필요합니다.

- **인증**: OAuth2 / 사내 SSO / 세션 쿠키 등과 연동하는 **백엔드 또는 BFF(Backend for Frontend)**.
- **이력·검수**: 살포·검수 결과를 저장하는 **DB**와 조회·필터용 **API**.
- **실시간**: WebSocket/MQTT 등으로 **선박 위치·살포 이벤트**를 푸시하는 **메시징·구독 서버**(선택).

## 프론트엔드 구성

| 구분 | 내용 |
|------|------|
| **런타임** | React 18, Vite 6 |
| **스타일** | Tailwind CSS 4, 전역 스타일 `src/styles/index.css` |
| **아이콘** | Lucide React |
| **진입점** | `src/main.tsx` — 로그인 여부에 따라 `LoginPage` 또는 `Dashboard` 표시 |
| **주요 화면** | `src/app/LoginPage.tsx`, `src/app/Dashboard.tsx` |

빌드 결과물은 정적 파일(`dist/`)이며, **정적 호스팅(Nginx, S3, 사내 웹 서버)** 만으로 배포 가능한 형태입니다.

## 화면 흐름 및 작동 방식

### 1) 로그인 화면 (`LoginPage`)

- 이메일·비밀번호 입력 폼이 있으나, **시연 모드**로 **비밀번호·계정 검증 없이** 제출 시 `onSuccess()`만 호출되어 대시보드로 전환됩니다.
- 카피: 「관리자 인증 · 내부망 전용」「대한민국 관공서 납품 시연 화면」 등 **납품·시연 맥락**을 전제로 한 문구입니다.

### 2) 관제 대시보드 (`Dashboard`)

- **좌측 사이드바**
  - 금일 살포·누적 건수·속도 등 **고정에 가까운 시연 수치**.
  - **선박 위치**(위도·경도·침로) — 주기적 클라이언트 갱신.
  - **종자 살포 이력** 목록 — 시각·좌표, 최신 건 강조.
  - **조회 기간**(시작일·종료일)으로 이력 필터.
  - **CSV 파일 저장** — 필터된 목록을 브라우저에서 **UTF-8 BOM** CSV로 다운로드(서버 저장 아님).

- **우측 지도 영역**
  - **현재 시각** 오버레이.
  - **살포 색상 안내** 버튼(우상단): 클릭 시 모달이 아니라 **버튼 바로 아래 패널**로 범례·설명 표시, **닫기** 또는 버튼 재클릭으로 접음.
  - 살포 점 색: **기록 시각 경과**에 따른 구간색 + **검수 불일치·누락**은 **검정** (`verificationMismatch` 플래그).
  - **휠 줌·플로팅 줌 버튼**, 항적·선박 아이콘·살포 점 SVG 렌더링.
  - 하단 **InfoCard**(활동 해역·살포 밀도·커버리지) — 시연용 텍스트.

- **선박·살포 시연 로직(요약)**
  - 선박은 미리 정의된 **웨이포인트**를 따라 이동합니다.
  - 일정 간격으로 현재 위치 근처에 **새 살포 점**이 추가되며, 최대 개수 제한으로 오래된 점은 잘립니다.
  - 초기 로드 시 **과거 시점이 섞인 시드 살포**와 **검수 불일치 1건** 등이 포함되어 색 구간·검정 점을 한눈에 볼 수 있게 합니다.

## 빌드·로컬 실행

```bash
npm install
npm run dev      # 개발 서버 — 기본 http://192.168.45.214:5111/ · 준비되면 기본 브라우저 자동 실행(끄기: VITE_OPEN=0, 열 URL 덮어쓰기: VITE_OPEN_URL)
npm run build    # 프로덕션 정적 빌드 → dist/
```

### Cursor / VS Code 안에서 미리보기

- 명령 팔레트(`Ctrl+Shift+P`) → **Tasks: Run Task** → **「심플 브라우저로 앱 열기 (Cursor 창 안)」** 를 실행하면 편집기 내 **심플 브라우저**로 `http://192.168.45.214:5111/` 가 열립니다. (`npm run dev`가 떠 있는 상태여야 합니다.)
- URL은 `.vscode/tasks.json`의 `simpleBrowser.api.open` 인자와 맞춰 두었습니다.

## 화면 캡처 스크립트 (Playwright)

- **스크립트**: `scripts/capture-submission.mjs`
- **명령**: `npm run capture:submission`
- **동작**: 지정 URL 접속 → 「관제 시스템 접속」 클릭 → 대시보드 대기 → 「살포 색상 안내」 열림 확인 → PNG 저장.
- **환경 변수**
  - `CAPTURE_URL` — 기본 `http://192.168.45.214:5111/`
  - `CAPTURE_OUT` — 기본 `submission-dashboard.png` (프로젝트 루트 기준 저장 경로)

캡처 전에 **개발 서버가 `CAPTURE_URL`과 동일 주소**에서 떠 있어야 합니다.

## 정리

| 질문 | 답 |
|------|-----|
| 백엔드가 있나요? | **없습니다.** 전부 브라우저 내 React 상태·시연 로직입니다. |
| 로그인은 안전한가요? | **시연용**이며 실제 검증·토큰 발급은 없습니다. |
| 데이터는 어디에 저장되나요? | **저장되지 않습니다.** 새로고침 시 초기 데이터로 돌아갑니다. |
| 지도는 실제 위성/도로맵인가요? | **아니요.** SVG 플레이스홀더이며, 문구상 외부 지도 API는 **향후 연동 예정**입니다. |

본 문서는 저장소의 현재 구현을 기준으로 작성되었으며, 이후 백엔드·지도·인증을 붙이면 해당 절을 별도 문서로 갱신하는 것을 권장합니다.
