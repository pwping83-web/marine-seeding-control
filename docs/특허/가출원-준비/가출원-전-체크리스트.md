# 가출원(빠른 출원) 전·후 체크리스트

> 절차·서식은 **특허청·특허로 최신 안내**가 우선이다. 아래는 **내부 실무용**이다.

## 출원 전

- [ ] 발명의 **명칭**·**요지**·**실시예**가 **실제 구현**과 크게 어긋나지 않는지 확인 (`변리사-제출용`·`관공서-제출용/docs/기술-요구사항-정의` 대조).
- [ ] **도면**: 아키텍처 PNG, 관제·항로 화면 캡처 준비 (`변리사-제출용/04`·`figures/` 참고).
- [ ] **발명자·출원인** 정보(성명·주소 등)를 출원 시스템에 넣을 값으로 정리 — **본 체크리스트 파일에는 적지 말 것**(로컬 메모).
- [ ] **선행기술**을 대략 KIPRIS 등에서 검색해 두면(선택) 이후 보정에 도움이 된다.
- [ ] 가능하면 **변리사 1회 상담**: “혼자 출원한 명세”의 빈칸·청구항 리스크를 줄인다.

## 출원 당일(특허로 등)

- [ ] 명세서 **전 부** 작성 여부(빈칸·「후补」만으로는 위험할 수 있음 — 제도 확인).
- [ ] **청구항** 최소 1항이라도 **구체**하게(장치/방법/시스템 중 택일·병기).
- [ ] 도면 **번호·간단한 설명**이 명세와 대응하는지.
- [ ] 출원 후 **접수증·출원번호**를 **`가출원-로컬메모.md`**에만 기록( Git 금지 ).

## 출원 후

- [ ] **우선일·출원일** 로컬 메모 백업(외장 디스크·암호 ZIP 등).
- [ ] 정식 보완·중간사건 기한이 있으면 **달력 알림** 설정(특허로·우편 통지 기준).
- [ ] 코드·명세 **어긋남**이 생기면 `출원-명세-합본-초안-복사용.md`와 레포 문서를 **같이 갱신**할 것.

## 하지 말 것

- 출원번호·접수 스캔을 **채팅·공개 이슈**에 붙이기.
- 아직 출원하지 않은 청구항을 **제3자에게 “등록됐다”**고 말하기.
