# 내가 보는 요약 — 특허·발명 자료 한 장 이해

## 이 시스템에서 “특허로 말하고 싶은 것”을 쉽게 말하면

1. **바다에서 종자를 뿌릴 때**  
   기상(풍·파고·시정 등)과 해역 정보를 바탕으로 **오늘 얼마나 뿌릴지**, **어느 구역이 점수가 좋은지**를 정하고, **암초·보호구역을 피하는 그리드 A\***로 항로를 짜되, **이동 스텝마다 풍향·풍속·돌풍으로 만든 배수**로 스텝 비용을 조절한다(기상 없으면 거리만). (코드: `seeding-plan-ai.ts`, `kma-weather.ts` 등)

2. **배에 달린 기기(아두이노 등)가 서버로 데이터를 보낼 때**  
   지도 API 키나 DB 비밀번호를 기기에 넣지 않고, **살포 한 번 한 번**과 **배 위치 궤적**을 **서로 다른 주소(Edge 함수)**로 보낸다. 그리고 **초당 요청 수·한 번에 보내는 양**을 종류마다 다르게 제한해 서버가 죽지 않게 한다. (코드: `telemetry-ingest`, `vessel-track-ingest`)

3. **살포한 지점만 잔뜩 찍혀 있을 때**  
   그 점들을 감싸는 **볼록 다각형(Convex Hull)**으로 대략적인 **면적(헥타르)**을 계산해 화면에 보여 준다. 정밀 측량은 아니고 **관제·보고용 참고값**이다. (`seeding-day-eval.ts`)

4. **오래된 살포와 최근 살포를 지도에서 구분**  
   기록 시각(`recorded_at`)으로 **며칠 지났는지**에 따라 마커 색을 바꾼다. 장기 복원 사업에서 이력을 보기 좋게 하려는 **화면 정책**이다. (`seed-drop-visual.ts`)

## 변리사에게 꼭 말해 두면 좋은 “솔직한 한 줄”

- **A* 항로:** **장애물 회피** + 각 칸 이동마다 **거리(km)**에 **풍향·풍속·돌풍**으로 만든 **배수**(역풍이면 비용↑)를 곱한다. **작업 한도·구역 점수·PID**는 같은 파일의 **다른 함수**로 같이 쓴다. 출원 문장은 이 구조에 맞추면 된다.

- **망 끊겼을 때 기기 안에 저장했다가 다시 보내기**는 문서에서 **권장**하는 설계이지, 모든 스케치에 완성된 형태로 들어 있다고 단정하면 안 된다.
