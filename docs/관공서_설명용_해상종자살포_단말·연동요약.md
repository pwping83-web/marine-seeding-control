# 해상 종자 살포 — 아두이노(GPS·LTE) 단말 연동 요약 (관공서·심사 설명용)

본 문서는 **배 위 아두이노(또는 동급 임베디드 단말)**로 GPS·LTE를 사용해 **종자 살포 위치**와 **선박 궤적**을 관제 시스템에 반영할 때, **보안·신뢰성·규제 관점에서 권장되는 구성**을 한 장으로 정리한 것입니다. (본 저장소 구현과 대응합니다.)

---

## 1. 원칙 한 줄

| 하지 말 것 | 권장 |
|------------|------|
| 지도 서비스(API 키)를 단말 펌웨어에 심기 | 단말은 **우리 백엔드(HTTPS)만** 호출하고, 지도·화면은 **관제 웹**이 담당 |
| DB(anon 키)에 단말이 직접 쓰기 | **Edge Function 등 게이트웨이** + **기기 비밀·레이트 리밋** 후 DB 반영 |
| 단일 채널에 살포·항적·명령 혼합 | **살포 이벤트**와 **선박 궤적**을 **논리·테이블·API 분리** |

---

## 2. 데이터 두 갈래 (베스트 프랙티스)

### (1) 종자 살포 “이벤트” (한 번 뿌렸다는 사실)

- **의미**: 살포 장치 동작 시점의 위치·시각·(가능하면) 구역·배치 ID.
- **권장**: 직전·직후 GPS를 2~3회 샘플링해 **평균/중앙값**으로 한 점(또는 근접 소수 점) 기록 — 파도·멀티패스로 인한 튐 완화.
- **본 저장소**: Supabase Edge **`telemetry-ingest`** → 테이블 **`seed_drop_records`** (헤더 `X-Device-Ingest-Key` + 본문 검증·레이트 리밋).

### (2) 선박 “궤적” (항해 중 위치를 계속)

- **의미**: 배가 어디로 이동했는지 **시간 순 궤적** (감시·사후 검증·재난 대응).
- **권장 주기**: **30초~2분** 등 운영 정책·데이터 요금·배터리와 합의 (과도한 초당 전송은 불필요·부하).
- **끊김 대비**: LTE 불안정 시 **단말 큐(플래시/SD)** + 재전송·백오프.
- **본 저장소**: Edge **`vessel-track-ingest`** → 테이블 **`vessel_track_points`**; 관제 웹에서 **「해상 기기(LTE) 궤적」** 토글로 주기 조회·지도에 주황 궤적 표시.

---

## 3. 보안·개인정보 (심사에서 먼저 보는 축)

1. **전송**: 단말↔서버 **HTTPS(TLS)** 만 사용. (패킷 스니핑 대응은 TLS + 네트워크 구간 설계.)
2. **비밀 관리**: 지도 API 키가 아니라 **수집 전용 비밀(`DEVICE_INGEST_SECRET`)** 은 **Supabase Secret**에만 두고, 단말에는 **펌웨어 내 보호**(유출 시 즉시 회전·폐기 가능하게 운영).
3. **남용 방지**: Edge에서 **요청당·IP당·전역 레이트 리밋**, 페이로드 크기·좌표 범위 검증 (악의적 대량 전송 완화). **분산 DDoS**는 WAF·CDN·망 방화벽과 **병행**한다고 설명.
4. **DB 접근**: 단말이 **anon 키로 테이블 직접 INSERT** 하지 않음 — **서비스 롤은 Edge 내부만**.
5. **저장 구간**: 민감 필드가 있으면 **애플리케이션·DB 암호화(AES 등)·KMS** 및 **공공 클라우드·ISMS-P** 요구에 맞춘 호스팅을 별도 명세.

---

## 4. 신뢰성·운영 (바다 현장)

| 항목 | 설명 포인트 |
|------|-------------|
| 시각 동기 | 가능하면 **GPS UTC 또는 NTP**로 `recorded_at` 일관성 확보 |
| 식별자 | `vessel_id`·살포 ID는 **함정·항차·일련번호 규칙**으로 유일·추적 가능하게 |
| 장애 | Edge·DB 장애 시 단말은 **로컬 큐 보존** 후 재전송 |
| 관제 화면 | 궤적은 **폴링 또는 Realtime**으로 갱신; “25분 이상 무신호” 등 **스테일 알림**으로 현장 이상 징후 표시(본 웹 UI 참고) |

---

## 5. 규제·표준 (한 줄 안내)

- **웹 접근성·호환성**: 관제 UI는 WCAG·브라우저 호환을 별도 점검표로 관리.
- **공공 클라우드**: 실제 공공 데이터면 **행안·기관 정책에 맞는 인증·전용망/CSAP 등**과 호스팅 위치를 명세에 포함.
- **공동·금융 인증서 로그인**: 브라우저 단독 완결이 아니라 **GPKI·OIDC/SAML 게이트웨이 + IdP 연동**이 통상 경로(본 프로젝트는 게이트웨이 URL 환경 변수로 확장 가능).

---

## 6. 현장 반입 체크리스트 (베스트 순서)

1. Supabase에 SQL 적용: **`004`**(레이트 리밋 RPC·접속 로그 상한) → **`005`**(`vessel_track_points`) → 기존 **`001`~`003`** 스키마와 정합.
2. Secret 설정 후 Edge 배포: **`telemetry-ingest`**, **`vessel-track-ingest`** (`DEVICE_INGEST_SECRET` 동일 사용 가능).
3. 단말 펌웨어: **HTTPS POST** + JSON 스키마 확정 + **주기·큐·재시도**.
4. 관제 `.env`: `VITE_SUPABASE_*`, 필요 시 **`VITE_VESSEL_LTE_ID`** / **`VITE_VESSEL_LTE_POLL_MS`**.
5. 시험: 육상에서 **소량 궤적·살포 이벤트** 전송 → 웹 지도·이력 반영 확인 → 해상 시험.

## 8. 심사 답변 예시 (한 문장)

> “단말에는 지도 키를 넣지 않고, LTE로 **인증된 수집 게이트웨이**에만 HTTPS로 **살포 이벤트**와 **선박 궤적**을 분리 전송하며, 서버에서 **레이트 리밋·검증** 후 DB에 기록하고 관제 웹이 **지도·이력**으로 표시합니다. 유출·남용에 대비해 비밀은 **서버 Secret 관리**와 **WAF·망 정책**을 병행합니다.”

## 참고 — 관제 웹(대시보드) 최신 요약

육상 **관제 웹**은 Leaflet 지도, **AI 기상 안전·8시간 타임라인**, 긴급 명령·SOS 시연, **항로·속도 패널(항상 표시)**, AI 살포 계획 탭, 관공서용 **고도화 모달** 등으로 구성되어 있습니다. **고도화(개발 예정)** 에는 **암초 센서로 탐지한 좌표를 기록·지도에 표시**하고, 향후 **공유 데이터를 바탕으로 항로·자율운항이 암초를 자동 회피**하는 방향이 포함되어 있습니다(실구현 전 기획·제안용). 세부는 **`docs/해양-종자-살포-관제-시스템-개요.md`** 의 「기능 개발 예정」절 및 **`docs/사용자매뉴얼_v1.5.md`** 를 참고하세요.
