# B2B SaaS 요금 안내 & 제안서 페이지 자동 구성

사용자가 제품·서비스의 요금 정보를 제공하면, 홈페이지·제안서·미팅 자료로 바로 사용할 수 있는 **다크 모드 1페이지 웹사이트**를 생성합니다.

## Step 1: 웹사이트 스킬 위임 (BLOCKING — 반드시 먼저 실행)

다른 작업을 시작하기 전에 `load_skill` 도구를 호출하여 내장 website 스킬을 로드합니다.

```
load_skill(skill_name="website")
```

website 스킬이 제공하는 `init_project`, `build_and_deploy_html_content`, `build_and_deploy` 등의 도구를 사용해야 합니다. HTML을 직접 파일로 쓰거나 `jupyter_execute`로 처리하지 마세요 — 배포 파이프라인은 website 스킬이 소유합니다.

website 스킬 로드 후, 아래 Step 2~5를 따릅니다.

---

## Step 2: 요금 정보 수집

사용자가 제공한 정보에서 아래 항목을 추출합니다. 값이 없으면 "별도 문의" 또는 "협의"로 표기하고, 추측으로 채우지 않습니다.

| 항목 | 설명 |
|------|------|
| **서비스명** | 제품·서비스 이름 (히어로 타이틀에 사용) |
| **한 줄 슬로건** | 요금의 핵심 메시지 (예: "월 50만 원부터, 설치는 필요할 때만") |
| **기본 요금** | 가장 낮은 월 구독료 · 기준 단위 (예: ₩500,000/월·1석) |
| **추가 요금** | 기본 단위 초과 시 요금 (예: +₩200,000/척) |
| **통화 & 기준일** | ₩ / $ / ¥ 등, 고지 날짜 |
| **포함 항목** | 구독료에 포함된 기능 목록 |
| **미포함 항목** | 별도 청구 항목 목록 (항목명 + 단가 + 설명) |
| **플랜 비교** | BASIC / STANDARD / PREMIUM 등 플랜별 가격·기능 |
| **연간 시뮬레이션** | 1년 총비용 예시 (1회성 + 구독 합산) |
| **연 계약 할인** | 선납 할인율 표 (선택) |
| **FAQ** | 자주 묻는 질문 5개 이상 |
| **CTA** | 문의처 (이메일·전화·URL) |
| **회사 정보** | 상호·대표자·사업자번호·주소 (푸터용) |

정보가 부족해도 즉시 작업을 시작합니다. 빠진 필드는 플레이스홀더로 표시하고 진행합니다.

## Step 3: 페이지 구성 규칙

### 3-1. 디자인 시스템 (다크 모드 고정)

```css
:root {
  --bg:           #0d1117;   /* 최외곽 배경 */
  --surface:      #161b22;   /* 카드·패널 배경 */
  --surface2:     #1e2530;   /* 테이블 헤더 */
  --border:       #2a3240;   /* 구분선 */
  --cyan:         #00e5ff;   /* 포인트 컬러 (브랜드 색상 변경 가능) */
  --cyan-dim:     rgba(0,229,255,.08);
  --cyan-border:  rgba(0,229,255,.22);
  --text:         #f0f6fc;   /* 본문 텍스트 */
  --muted:        #8b949e;   /* 보조 텍스트 */
  --warn:         #f0a500;   /* 별도·경고 강조 */
  --green:        #3fb950;   /* 포함·긍정 강조 */
}
```

> 브랜드 색상이 따로 있으면 `--cyan` 값을 해당 색으로 교체합니다.

폰트: `Pretendard` (CDN: `https://cdn.jsdelivr.net/gh/orioncactus/pretendard/dist/web/static/pretendard.css`)  
레이아웃: `max-width: 960px`, 중앙 정렬, `padding: 48px 24px 80px`  
반응형: 모바일(640px 이하)에서 2단 그리드 → 1단으로 전환

### 3-2. 섹션 순서 (반드시 이 순서 준수)

| # | 섹션 | 핵심 컴포넌트 |
|---|------|--------------|
| 1 | **히어로** | 서비스명, 슬로건, 기본 요금 pill, 기준일 |
| 2 | **핵심 요금** | 카드 그리드 (구독·추가·별도 항목), 콜아웃 박스 |
| 3 | **포함 / 미포함** | 2단 비교 박스 (✓ 포함 vs ✕ 별도) |
| 4 | **상세 단가표** | 별도 항목 전체 테이블 (항목·설명·단가) |
| 5 | **플랜 비교** | 2~4단 플랜 카드 (BASIC·STANDARD·PREMIUM 등) |
| 6 | **연간 시뮬레이션** | 1년 총비용 테이블 (1회성 + 구독 × 12) |
| 7 | **연 계약 할인** | 선납 할인 테이블 (있을 경우, 없으면 생략) |
| 8 | **이런 고객에게** | 타깃 고객 카드 3개 (선택, 정보 있을 때만) |
| 9 | **FAQ** | 아코디언 또는 카드 리스트 |
| 10 | **CTA** | 이메일·전화 버튼, 다음 단계 안내 |
| 11 | **푸터** | 회사 정보, 고지 문구, 버전·기준일 |

### 3-3. 컴포넌트 세부 규칙

**히어로**
- 상단에 브랜드 뱃지 (점멸 애니메이션 포함)
- 기본 요금은 큰 숫자(36px+)로 강조
- 부가세 별도 여부를 항상 명시

**핵심 요금 카드**
- 구독 요금 카드: `--cyan-dim` 배경 + `--cyan-border` 테두리 (강조)
- 별도 항목 카드: `--warn` 색상 배지 (`별도` 레이블)
- 카드 아래에 콜아웃 박스로 핵심 주의사항 1문장 강조

**포함/미포함 박스**
- 좌: `--green` 컬러, ✓ 불릿
- 우: `--warn` 컬러, ✕ 불릿
- 각 항목은 간결하게 (1줄 이내)

**테이블**
- `border-radius: 12px`, `overflow: hidden` 적용
- 헤더 배경: `--surface2`, 헤더 텍스트: `--cyan`
- 짝수 행: 미세 배경 (`rgba(255,255,255,.015)`)
- 금액 셀: `--cyan` + `font-weight: 700`
- "협의" 또는 "별도" 셀: `--warn` + `font-weight: 700`

**플랜 카드**
- 추천 플랜(중간 티어)에 `featured` 클래스: `--cyan-border` 테두리 + `--cyan-dim` 배경
- 플랜명은 `text-transform: uppercase`
- 가격, 기준 단위, 구분선, 기능 목록 순서로 배치

**연간 시뮬레이션 테이블**
- 합계 행: `border-top: 2px solid var(--cyan-border)`, `font-weight: 800`, 금액 `--cyan`
- 주석으로 "추가 비용 없을 때 기준" 명시

**FAQ**
- Q: `font-weight: 700`, 본문 컬러
- A: `--muted` 컬러, `<strong>` 태그로 핵심 키워드 강조

**CTA 박스**
- `linear-gradient(135deg, rgba(--cyan,.1), rgba(--cyan,.04))` 배경
- 버튼: 주 버튼(채움) + 보조 버튼(아웃라인)

## Step 4: 빌드 방법

### A. Claude Code + website 스킬 (우선)

website 스킬의 **Lite 모드**로 진행합니다.

1. `init_project(project_mode="static_html", website_category="landing_page")` 호출
2. 완성된 HTML을 `build_and_deploy_html_content(content=<html>)`에 전달
3. 빌드 실패 시 에러 메시지를 읽고 수정 후 재시도 (최대 3회)

### B. Cursor / 이 저장소 (website 스킬 없을 때)

1. 요금 원문: `docs/제품-운영/구독-요금제-소개페이지.md` · 내부: `ENX-단가표-영업대응-가이드.md`
2. 산출물: **`public/pricing.html`** 단일 파일 (인라인 CSS·JS, Pretendard CDN)
3. `npm run dev` 후 `/pricing.html` 로 확인 · Vite 정적 배포 시 동일 경로

> Lite 모드는 단일 HTML입니다. 외부 JS는 CDN·인라인만 사용합니다.

## Step 5: 출력 품질 기준

배포 후 다음을 확인합니다:

- [ ] 사용자가 제공한 모든 요금 수치가 정확히 반영되어 있음
- [ ] "협의" / "별도" 항목이 실제보다 부풀려지지 않음 (과장 금지)
- [ ] 부가세 별도 여부가 히어로·테이블·푸터 3곳 이상 명시됨
- [ ] 모바일(640px)에서 레이아웃 깨짐 없음 (미디어쿼리 적용)
- [ ] 외부 AI 생성 이미지 사용 금지 — 텍스트·아이콘·CSS만 사용
- [ ] CTA 버튼에 실제 이메일 또는 전화번호가 들어가 있음
- [ ] 푸터에 기준일·버전·고지 문구 포함

## 주의사항

- **금액 수치는 사용자가 제공한 값만 사용합니다.** 없는 수치를 임의로 만들지 마세요.
- **"포함" 항목을 "미포함"으로, 혹은 그 반대로 뒤바꾸지 않습니다.** 요금 안내는 계약의 근거가 됩니다.
- 포인트 컬러(`--cyan`)는 사용자가 별도 브랜드 색상을 제공하면 교체합니다.
- 플랜이 1개뿐이면 "플랜 비교" 섹션을 생략하고 핵심 요금 섹션을 확장합니다.
- 할인 정보가 없으면 "연 계약 할인" 섹션을 생략합니다.
