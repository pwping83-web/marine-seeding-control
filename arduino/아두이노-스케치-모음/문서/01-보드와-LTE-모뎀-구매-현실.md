# 01. 보드·LTE 모뎀 — 현실적으로 무엇을 살까

“아두이노”라고만 하면 부품이 너무 많습니다. **해상 + GPS + 인터넷(LTE)** 를 동시에 만족시키려면, 현장에서는 아래 조합이 **가장 현실적**입니다.

---

## 1. 왜 “클래식 아두이노 Uno”만으로는 부족한가

- **Uno / Nano**: 메모리·속도가 작고, **HTTPS(암호 웹통신)** 를 안정적으로 하기 어렵습니다. LTE 모듈을 UART로 붙이는 예제는 있으나, TLS·JSON·동시에 GPS 파싱까지 하면 빡빡합니다.
- **권장**: **ESP32** 계열(또는 동급 **32비트 MCU + 충분한 RAM**)을 **메인 두뇌**로 쓰고, GPS·LTE는 **주변 모듈**로 붙입니다.  
  (상표가 “Arduino”가 아니어도 되고, **Arduino IDE로 같은 방식**으로 빌드하는 보드면 됩니다.)

---

## 2. 보드(메인 MCU) — 이렇게 고르기

| 등급 | 예시 제품군 | 특징 | 대략 감 |
|------|-------------|------|--------|
| 입문 | **ESP32-DevKitC**, ESP32-WROOM | 저렴, WiFi 내장, 문서 많음. LTE는 **별도 모듈** UART 연결 | 만 원대~ |
| 한 장에 모뎀까지 | **LilyGO T-SIM7600**, **T-A7670**, **TTGO T-Call** 등 | ESP32 + SIM 슬롯 + GNSS 안테나 단자까지 보드에 있음 | 3~8만 원대 (변동) |
| 산업용 | 박스형 RTU, 방수 단말기 | 방수·전원·인증 — 예산 크게 잡음 | 별도 견적 |

**현장 추천 한 줄**: 처음이면 **ESP32 개발보드 + LTE 모듈 키트** 또는 **LilyGO SIM7600 계열 한 장** 중 하나를 고르세요.

---

## 2-1. GPS + LTE **내장(한 보드)** 으로 살 때 (구매 예정이 이쪽일 때)

말씀하신 것처럼 **GNSS(GPS) 안테나 단자 + SIM 슬롯 + LTE 모뎀**까지 한 PCB에 올라간 **ESP32 계열 개발보드**(예: LilyGO **T-SIM7600** / **T-A7670** / **T-Call** 등, 제품명은 시즌마다 바뀜)를 고르면 됩니다.

| 체크 | 설명 |
|------|------|
| **밴드·유심** | 아래 **§4 LTE 모듈** 절과 동일 — **한국 통신사 밴드**와 **IoT/M2M 유심**을 구매 전에 표로 대조 |
| **전원** | 일체형도 송신 순간 **전류 폭**은 큼 — USB만으로는 불안정할 수 있어 **배·외부 DC** 설계 권장 |
| **안테나** | **LTE 메인/분할** + **GNSS(GPS)** 안테나를 빠뜨리지 말 것(키트에 동봉되는지 확인) |
| **소프트웨어** | 본 저장소 스케치 `esp32_marine_telemetry.ino` 는 기본이 **WiFi + `WiFiClientSecure`** 로 HTTPS 를 보냄. **LTE만** 쓸 때는 보드·모뎀 예제처럼 **TinyGSM 등으로 셀룰러 데이터 연결 후 HTTP(S)** 경로로 바꾸는 작업이 필요(같은 JSON·같은 Edge URL 이면 됨) |

**한 줄**: “아두이노” 상표보다 **ESP32 + SIM7600급 일체 보드**가 해상 단말 프로토타입에 가장 많이 쓰입니다.

---

## 3. GPS(GNSS) 모듈

| 항목 | 설명 |
|------|------|
| 종류 | **NEO-6M / NMEA 출력** 저가 모듈부터 **u-blox M8/M9** 까지 다양 |
| 안테나 | **패치 안테나**는 금속 덮이면 망가짐 → **막힘 없는 상단** + 가능하면 **외장 유동 안테나** |
| 출력 | 보통 **9600 baud**, `$GPGGA` `$GPRMC` 등 **NMEA 문장** (스케치에서 파싱) |
| 정확도 | 개방 하늘 **수 m~10m급**. 옆 구조물·선박 상부 설치 위치에 따라 다름 |

---

## 4. LTE 모듈 — 무엇을 사야 하나

**한국에서 쓰려면** 다음이 **구매 전 필수**입니다.

1. **통신사(SKT/KT/LGU+)·MVNO** 에서 **IoT/M2M 데이터 유심** 가능 여부  
2. 모뎀이 지원하는 **LTE 주파수 밴드**가 그 통신사와 **겹치는지** (예: B1, B3, B5, B7, B8 등 — 제품 스펙 시트에 표기)  
3. **Cat-1 / Cat-4** 일반 LTE vs **LTE-M / NB-IoT** — 요금·커버리지 정책이 다름 (현장은 통신사에 확인)

**자주 쓰이는 모듈 계열 (예시 이름만 — 구매 전 밴드·유심 확인)**  

| 계열 | 비고 |
|------|------|
| **SIM7600** (멀티밴드 LTE) | 키트·쉴드 많음, 전류 소비 큼 — **전원(배터리·DC-DC)** 여유 있게 |
| **SIM7080 / A7670** | Cat-M/NB 위주 제품도 있음 — **해상 일반 LTE** 와 다를 수 있음 |
| **BG96** | LTE-M/NB 등 — 역시 **요금제·밴드** 확인 |

**주의**: 제목에 “4G”만 있어도 **한국에서 안 잡히는 밴드**일 수 있습니다. **판매 페이지의 주파수 표**와 **통신사**를 꼭 대조하세요.

---

## 5. 전원·기계 (배 위 현실)

| 항목 | 권장 |
|------|------|
| 전압 | 대부분 **3.3V 로직** — 5V TTL 모뎀은 **레벨 시프터** |
| 전류 | LTE 송신 순간 **수백 mA~1A 이상** — Uno의 3.3V 레귤레이터로는 부족할 수 있음 → **별도 DC-DC** |
| 방수 | **IP65 이상 박스**, 케이블 그랜드, 응축 방지  
| 온도 | 동해·동절 시 **사업용 등급** 모듈·박스 검토 |

---

## 6. “아두이노 공식 보드”만 고집할 필요는 없음

관공서 설명용으로는 **“현장 단말(임베디드 컴퓨터)”** 이라고 부르고,  
**Arduino IDE + ESP32 보드** 조합이 **가장 문서·예제가 많다**고 말해도 됩니다.
