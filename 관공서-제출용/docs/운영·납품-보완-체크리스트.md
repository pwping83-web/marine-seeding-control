# 해양 종자 살포 관제 — 운영·납품 보완 체크리스트

**용도:** 관공서 제출·과제 심사·납품 전에 **현재 저장소의 한계**를 메우기 위한 **작업 순서**와 **역할 분담**을 정리합니다.  
**전제:** 본문은 법률·계약 자문이 아니며, 제출처·사업 요구서에 맞게 수정·삭제하세요.

---

## 0. 제출 직전 — 문구·화면 점검 (심사·열람 관점)

이메일·USB·포털로 **URL과 문서를 보내기 전**에 아래를 다시 확인하십시오.

| 점검 | 할 일 |
|------|--------|
| **기관 명칭·납품 단정** | 화면·PDF에 **「○○부 납품 완료」**처럼 사실과 다른 문구가 없는지. (본 저장소는 **시연·제안용** 전제 문구로 정리함.) |
| **연락처·개인정보** | 매뉴얼 하단·캡처에 **실제 번호·주소**만 남기고, 더미·예시·미기입 표시를 제거했는지. |
| **「AI 참고」「자동 생성」 류** | 제출 PDF·md에 **모델 대화·참고 문항** 같은 표현이 없는지. 공고 모니터링 파일은 **내부 참고** 문구로 시작하는지. |
| **지도·데이터 설명** | 지도가 SVG인지 위성인지 **실제 구현과 일치**하는지(매뉴얼 FAQ는 Leaflet 타일 기준으로 맞춤). |
| **버전·보안 표기** | `SECURE` 등 **근거 없는 마케팅 단어** 없이, 빌드 번호만 표기하는지. |
| **외부 URL** | Vercel·LAN 주소가 **제출처에서 접속 가능한지**, 불필요한 외부 링크가 없는지. |

## 1. 이번에 저장소에서 반영한 보완 (자동/코드)

| 항목 | 내용 |
|------|------|
| **접속 로그·지리정보** | `LoginPage.tsx`에서 Supabase 접속 로그 기록 시 `fetchClientAccessGeo` import 누락을 수정해, DB 로깅이 켜진 환경에서 런타임 오류가 나지 않도록 함. |
| **로그인 화면 안내** | 시연용 접속·로컬/공개 URL 차이·운영 시 SSO·망분리 권고·「공식 누리집 아님」 문구. 기본 이메일 칸 비움·하단 `빌드 1.5.0` 등 과장 표기 제거. |
| **매뉴얼·고도화 문구** | 미승인 **해양수산부 납품** 단정 제거, 지도 FAQ를 **Leaflet 타일** 설명으로 정정, 고도화 모달의 과장된 한 줄 슬로건을 **중립적 목표 문장**으로 교체. |
| **공고 모니터링 md** | `99_…` 상단 안내를 **내부 참고·원문 우선**으로 통일(`grant-watch/run.mjs` 동일). |

추가 구현(백엔드·SSO·ISMS 등)은 **아래 순서에서 사업주·수행사**가 진행합니다.

## 2. 사업주(귀하)가 할 일 — 권장 순서

### 0단계 — 요구서 고정 (반드시 먼저)

1. **입찰공고·과제요구서(RFP)** PDF에서 다음을 표로 추출합니다.  
   - 인증·망·개인정보·로그·가용성·장비·시험해역·데이터 귀속·유지보수
2. 본 문서 **§4 빈 표**에 항목을 옮겨 적고, 각 칸에 **충족 / 부분 / 미충족**만 표기합니다.  
3. **미충족·부분**만 개발 범위(추가 견적)로 산정합니다.

---

### 1단계 — 시연·제출을 “설명 가능한 상태”로 (1~2주 내)

4. **캡처·제출 패키지**  
   - `pnpm run capture:gov` 로 `관공서-제출용/captures/` 생성  
   - 상위 `관공서-제출용/README.md` 에 따라 PDF 묶음 등 제출처 형식 맞춤  
5. **기술 설명 자료 병행**  
   - `관공서-제출용/docs/관공서_설명용_해상종자살포_단말·연동요약.md`
   - `docs/제품-운영/해양-종자-살포-관제-시스템-개요.md`
   - `관공서-제출용/docs/세부계획서-샘플테스트-아두이노-웹연동-단계별시험계획.md`
6. **시연 URL·내부망 주소**를 제출물 표지·본문에 **한 곳에만** 명시하고, 불필요한 외부 노출은 줄입니다.

---

### 2단계 — 운영 환경·계정 (납품 직전)

7. **호스팅**  
   - 정적 프론트: 사내 Nginx·S3·전용 웹 등 (저장소는 `dist/` 정적 빌드)  
   - **도메인·TLS 인증서**·갱신 담당자 지정  
8. **Supabase(또는 대체 백엔드)**  
   - 프로젝트 분리: **시연용 / 운영용**  
   - `.env.example` 기준 키·Secret·Edge(`telemetry-ingest`, `vessel-track-ingest`) 배포  
   - **RLS·서비스 롤** 점검: 단말·클라이언트가 anon으로 테이블 직접 쓰지 않는지 재확인  
9. **로그인**  
   - 공개 배포 시: **Supabase Auth** 실계정 또는 **기관 SSO(OIDC/SAML)** 연동 설계  
   - `VITE_GOVT_CERT_LOGIN_GATEWAY_URL` 등은 기관 보안 담당과 협의 후 설정  
10. **기상·AI 키**  
    - `VITE_KMA_SERVICE_KEY`, `VITE_GROQ_API_KEY` 등: **과제에서 허용하는 범위** 내에서만 사용·보관

---

### 3단계 — 현장·단말·통신 (실증 기간)

11. **아두이노·LTE 모듈**  
    - `arduino/docs/` 및 예제 스케치와 실제 보드·SIM·안테나 사양 맞춤  
12. **해상 시험**  
    - 연못→호수→강→바다 등 `세부계획서` 단계에 맞춰 **기록 항목·주기·실패 시나리오** 정리  
13. **데이터 품질**  
    - GPS 멀티패스·LTE 끊김 시 단말 **큐·재전송** 동작 확인 (`관공서-제출용/docs/관공서_설명용_해상종자살포_단말·연동요약.md` 권장사항)

---

### 4단계 — 보안·개인정보·감리 (요구서에 따라 병행)

14. **개인정보·위치정보**  
    - 수집 목적·보관 기간·파기·제3자 제공 여부를 **내부 대장·동의서**에 반영  
15. **보안 점검**  
    - 취약점 스캔, WAF·레이트리밋, 관리자 계정 분리  
16. **등급화·ISMS-P 등**  
    - 해당 시 **별도 감리 체크리스트**에 맞춰 증적 폴더 구성 (본 문서로는 대체 불가)

---

### 5단계 — 유지보수·인수

17. **장애 대응**  
    - 담당자·에스컬레이션·RTO/RPO 목표(요구서에 있으면 그에 맞춤)  
18. **인수인계**  
    - 소스 저장소 접근권, Supabase 대시보드 권한, DNS·인증서, `.env` 보관 위치(비밀금고 등)
