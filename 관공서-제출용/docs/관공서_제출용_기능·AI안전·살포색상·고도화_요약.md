## 1. 현재 구현된 주요 기능 (웹 관제)

| 구분 | 내용 |
|------|------|
| **실시간 관제·작업 계획** | 지도 기반 관제 화면과 작업 계획 화면 전환, 기상·선박 상태·신호 송신 UI |
| **작업 계획 기상** | 7일 카드에 단기 합성 + (선택) **기상청 중기예보**로 날짜별 풍파고 보강 |
| **금일 항적·보고** | 모달에서 요약·CSV·**PDF 출력**(브라우저 인쇄), 살포점 3개 이상 시 **구역 추정 면적(ha)** |
| **선박 신호(시연)** | 귀항·살포 시작/중지·위치 보고 등 지령 버튼, 전송 시연 표시 |
| **살포·투하 표시** | 「살포 시작」과 연동된 시각적 강조, 「투하(종자 방류)」는 살포 활성 상태와 연동 |
| **종자 살포 이력** | 구역·시각·위도·경도 목록, 기간 필터, **CSV 저장·CSV 불러오기**, **건건별 삭제(휴지통)** |
| **금일 작업 보조(AI)** | 사이드바에서 모달 호출 — **로컬 규칙 기반 추천**과, 설정 시 **Groq API를 이용한 짧은 작업 요약** 병행 가능, **현장 메모**는 브라우저에만 저장(로컬) |
| **고도화 계획 안내** | 관제 UI 내 「무인화 고도화」 모달 — 단계별 비전·CCTV·원격조종·암초회피·안전 전환 등 **계획 수준** 설명(개발 예정 항목 명시) |

※ 실제 해상 운용·승인·통신 주파수·선박 안전 규정은 별도 법·지침을 따릅니다. 본 시스템은 **관제·기록·시연·실증 설계**를 지원하는 소프트웨어 측면을 다룹니다.

## 2. AI·「안전」 관련 (기상·작업 보조)

### 2.1 AI 기상 안전(사이드바)

- 기상청 API 등으로 받은 **풍속·파고·시정** 등과 **시간대별 점수**를 바탕으로, 화면에 **안전 / 주의 / 긴급** 수준을 표시합니다. **사이드바·지도 하단 타임라인·작업 권고**는 긴급 임계와 단기예보 첫 시간대 판정을 **더 보수적으로 합친 표시 등급**으로 통일합니다.
- **긴급** 등 위험 구간에서는 「즉시 회항 권고」 등 **운항 판단 보조 문구**를 강조합니다(최종 판단은 선장·관제 책임자).
- 선택적으로 **Groq** 등 외부 LLM으로 **날씨·위험 요약 문장**을 생성할 수 있으나, **API 키 미설정 시에는 요약 생성을 생략**하고 규칙 기반 표시만 사용할 수 있습니다.

### 2.2 금일 작업 보조(AI 모달)

- 당일 기상·안전 수준·로컬 추천을 한 화면에 모아 **작업 시간·작업량·유의사항** 형태로 제시합니다.
- **Groq** 사용 시: 위 입력과 **현장 메모**를 프롬프트에 반영한 **짧은 브리핑**을 받을 수 있으며, **근거·주의** 문구를 함께 표시하도록 UI에 여지를 두었습니다.
- **개인·민감 메모**는 서버로 필수 전송되지 않도록, 기본은 **브라우저 로컬 저장**만 사용합니다.

## 3. 살포 색상·저장 방식

### 3.1 지도 위 살포 점 색(시간 경과)

- 지도 마커 색은 **「기록 시각(recorded_at)이 얼마나 지났는지」**에 따라 구간을 나누어 표시합니다(예: 최근 약 45일 이내 → 진한 적계열, 시간이 지날수록 주황·분홍·회색 톤으로 완화).  
- **범례**에서 구간 의미를 확인할 수 있습니다.
- **검수 불일치 시연**용으로 별도 표시가 있는 점은 시연·교육 목적입니다.

### 3.2 사이드바 이력 목록 색

- 이력 **목록 행**의 왼쪽 강조선은 **UI 통일(청록 톤)** 을 위해 지도 연령색과 **별도**로 두어, 상단 패널과 이질감이 없도록 했습니다. **지도 색의 의미(연령)** 는 지도·범례에서 유지합니다.

### 3.3 데이터 저장

| 저장 위치 | 용도 |
|-----------|------|
| **브라우저 `localStorage`** | 키 `marine-seed-drops-v1` — 인터넷·DB 없이 시연할 때 **살포 이력 최대 약 500건**까지 JSON으로 보관 |
| **Supabase(선택 연동 시)** | 테이블 `seed_drop_records` 등 — 다중 단말·관제실 공유 시 **upsert·조회·삭제 API**와 연동 가능(환경 변수로 활성화) |

- **CSV**: 이력을 파일로보내거나, 검증용으로 다시 불러와 목록을 **덮어쓰기** 할 수 있습니다.
- **삭제**: 목록의 삭제 버튼으로 로컬에서 제거하며, DB 연동 시에는 **서버 삭제 성공 후** 목록에서 제거되도록 처리합니다.

## 4. 고도화 계획(로드맵 요약)

관제 웹 내 「**해양 무인 살포 시스템 고도화 계획**」모달에 다음과 같이 **3단계**로 정리되어 있습니다. **2·3단계는 계획·목표**이며, 일부는 개발 예정입니다.

### 1단계 — 현재 운영·구현 중심(요약)

- 기상·예보와 규칙 기반 **위험도·출항 판단 보조**  
- 관제탑 ↔ 선박 **신호·요약 리포트**  
- **살포 위치 추천·경로(A\*)**, **PID 속도 제어 시연**, **살포 이력 CSV**  
- 아두이노 등 단말과의 연동은 **별도 문서(단말·연동요약)** 의 게이트웨이·보안 원칙을 따름  

### 2단계 — 개발 예정(계획)

- **CCTV 실시간 관제**, **원격 조종(RC)**, **AI 자율 ↔ 수동 전환**, LTE+위성 이중망 등  
- **암초·저수심 센서 → 좌표 기록·지도 경고·공유 레이어·자동 회피** 등은 **개발 예정**으로 명시  

### 3단계 — 장기 목표

- 다중 선박, 공공 해양정보 연계·검증, **중대재해 예방 관점의 공정 설계** 등을 **정책·R&D 방향**과 함께 서술
