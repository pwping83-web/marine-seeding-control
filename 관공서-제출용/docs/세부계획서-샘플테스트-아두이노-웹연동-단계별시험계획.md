# 세부계획서(샘플) — 아두이노·관제 웹 연동 단계별 시험

본 문서는 **해양 종자 살포 관제 시스템**에 대해, 관공서·사업 설명·실증 계획 제출 시 활용할 수 있도록 **「연못 → 호수 → 강 → 바다」** 순으로 **위험도·통신 부담을 낮추며** 아두이노(또는 동급 단말)와 **관제 웹**을 점진적으로 검증하는 **샘플 시험 절차**를 정리한 것입니다.  
본 저장소의 **웹 대시보드**, **`arduino/`** 스케치, **`docs/관공서_설명용_해상종자살포_단말·연동요약.md`** 에 기술된 수집·보안 원칙과 **합치되도록** 작성하였습니다.

---

## 1. 문서 목적·적용 범위

| 구분 | 내용 |
|------|------|
| 목적 | 단말(GPS·입출력)과 관제 웹의 **연동·기록·비상 연동**을 단계적으로 입증하고, 제출용 설명 자료(캡처·개요)와 **같은 스토리라인**으로 정리한다. |
| 적용 | **육상·소수역**에서의 기능 시험을 우선하고, 해상 본운용 전 **검증 완료 조건**을 명시한다. |
| 비적용 | 본 문서만으로 **선박안전법·해사법 등 개별 인허가**를 대체하지 않는다. 실제 해상 시험은 기관·선사 절차를 따른다. |

## 2. 시험 원칙(요약)

1. **수역 단계**: **연못 → 호수 → 강(하류·하구) → 연안·바다** 순으로 확대한다.  
2. **기지(항구) 좌표**: 각 단계마다 **「이번 시험에서 선박이 출·입하는 기준점」**을 위·경도(또는 시험장 내 로컬 좌표계)로 **문서에 고정 기입**하고, 관제 웹·단말 설정(`VITE_VESSEL_LTE_ID` 등)과 **일치**시킨다.  
3. **기상·비상**: 관제 웹의 **기상·AI 안전 단계** 또는 **시연 모드**와 연계하여 **비상 알림·부저(릴레이)** 동작을 단계별로 기록한다.  
4. **1차 원격 조종**: **사무실(육상)**에서 **RC 수준**(방향·정지 등 제한 명령)으로 검증하고, CCTV·고속 링크는 **고도화 계획 2단계**로 구분한다.  
5. **웹 기록**: **선속(또는 대표 속도)**, **살포(이벤트) 건수·위치**, **궤적(위치 보고)** 가 단계마다 관제 화면·DB(또는 로컬 기록)에 **남는지**를 확인 항목으로 둔다.

## 3. 단계별 시험 정의(샘플)

아래 표는 **샘플**이며, 실제 연못·호수 명칭·좌표는 **시험 전에 기입·치환**한다.

| 단계 | 수역(개념) | 통신·단말(권장) | 가상 선박 기지(항구) 좌표 | 관제 웹에서 확인할 항목 | 비상·부저 시험 | 1차 원격(RC) |
|:----:|------------|-----------------|---------------------------|-------------------------|----------------|--------------|
| **0** | 사무실 책상 | USB 전원·시리얼 또는 Wi-Fi AP 내 통신 | **기준점 A** (예: 시험장 정문 앞 고정 좌표) | 대시보드 접속, 지도·이력 패널 표시 | 시연 모드로 **경보 문구·토스트** 확인 | 미적용(화면만) |
| **1** | **연못** | Wi-Fi 또는 LTE 테더링, GPS 실수신 또는 고정 모크 | **기준점 B** (연못 부근 지정 1점) | **위치 보고** 후 궤적·좌표 갱신, **선박 신호** 로그 | 기상 임계 초과 시 **웹 경보** + 단말 **부저(릴레이)** ON 조건 시험 | 사무실에서 **제한 명령**(예: 정지·호출)만, **저속·단거리** |
| **2** | **호수** | LTE(내부망 또는 공용), 배터리·방수 준비 | **기준점 C** (호수 선착장 등) | **살포 이벤트**·**속도 표시**·이력 CSV 연계 | 단계 1과 동일 패턴, **지연·끊김** 1회 이상 재현 후 복구 | RC: **방향·속도 단계** (고도화 계획상 개발 예정 영역은 **시뮬 UI**로 대체 가능) |
| **3** | **강(하류·하구)** | LTE, 안테나 높이·접지 점검 | **기준점 D** | 궤적 주기·**스테일(무신호)** 알림 | **귀항 명령** 수신 시 웹·단말 반응 | 육상 관제에서 **긴급 정지** 시나리오 |
| **4** | **연안·바다** | 해상용 안테나·인허가 반영 | **기준점 E** (실제 항구) | 본운용과 동일 **Edge 수집·DB** 경로 | 운영 매뉴얼대로 **이중 확인** | 2단계(영상·저지연)는 **별도 사업**으로 구분 |

## 4. 위치값 지정과 “선박·항구”의 대응

1. **관제 웹**: 테스트 시 **시뮬 모드** 또는 **실 GPS 모드** 중 하나를 선택하고, **「이번 시험의 항구」** 좌표를 내부 메모·시험 기록서에 명시한다.  
2. **단말**: 펌웨어 또는 설정에 **동일 좌표계(WGS84)** 로 기준점을 넣거나, 시험 0단계에서는 **고정 위치 모크**를 허용할 수 있다(연못·호수에서 GPS 다중경로 완화 목적).  
3. **본 저장소 연동**: 살포·궤적의 **서버 반영** 구조는 **`docs/관공서_설명용_해상종자살포_단말·연동요약.md`** 의 **이벤트 vs 궤적 분리** 원칙을 따른다. 육상 시험에서는 **로컬 전용 모드**(`LOCAL_RECORDING_ONLY` 등)와 **Supabase 연동**을 단계적으로 전환할 수 있다.

## 5. 기상 변화에 따른 비상 알림·부저 시험

| 순서 | 내용 | 합격 기준(샘플) |
|------|------|-----------------|
| 1 | 관제 웹에서 **풍속·파고 등** 임계를 넘는 시나리오(실측 또는 시연)를 발생시킨다. | 화면에 **주의/긴급** 단계 전환 및 **상단 자막·토스트**가 표시된다. |
| 2 | 단말에 **부저(릴레이)** 가 연결된 스케치(`arduino/sketches/` 참고)를 올린다. | `ship_command_logs` 등 **명령 채널** 또는 **폴링 API** 설계에 맞게 **비상 명령** 수신 시 부저가 동작한다(시험 기록에 전압·회로 사진 첨부 권장). |
| 3 | 기상이 완화된 것으로 가정하고 **해제·확인** 절차를 밟는다. | 웹·단말 모두 **정상 상태**로 복귀한다. |

※ 실제 소음·주변 민원이 있는 **연못·호수**에서는 **저음량·짧은 펄스** 또는 **LED 대체**로 대체할 수 있으며, 시험 계획서에 **대체 방법**을 적는다.

## 6. 1차 RC(사무실·육상) 원격 조정 시험

| 항목 | 샘플 내용 |
|------|-----------|
| 목적 | **“관제실에서 제한된 조작이 단말·웹에 반영되는가”** 를 연못·호수 단계에서 먼저 검증한다. |
| 범위 | **방향·정지·호출** 등 저위험 명령. **고속·장거리·무인 항해**는 본 문서 범위 밖. |
| 구현 대응 | 관제 웹의 **고도화 모달(원격 조종 탭)** 은 **개발 예정**으로 표시되어 있으므로, 1차 시험에서는 **선박 신호 송신**, **비상 패널**, **시뮬 조작**으로 **동등한 검증 목적**을 달성한다고 설명할 수 있다. |
| 기록 | 시험일시, 조작자, 명령 종류, 웹 캡처, 단말 로그(가능 시). |

## 7. 배 속도·살포량·위치의 웹 기록(연못·호수부터)

| 기록 항목 | 연못·호수에서의 확인 포인트 | 본 저장소 참고 |
|-----------|------------------------------|----------------|
| **속도** | 시뮬 속도 또는 GPS 유도 속도가 **대시보드 요약·지도**에 반영되는지 | `Dashboard` 요약 타일, 시뮬/실GPS 모드 |
| **살포량(건수)** | 살포 이벤트 추가 시 **이력 건수·라벨**이 누적되는지 | `seed_drop_records` / 로컬 모드 |
| **위치** | **위치 보고** 또는 궤적 수집 후 **지도·좌표**가 일치하는지 | `vessel_track_points`, 위치 보고 버튼 |

**캡처 제출**: `관공서-제출용/README.md` 의 **`pnpm run capture:gov`** 절차로 **대시보드·작업 계획·매뉴얼** 등을 생성하고, 본 시험 계획서 **부록 번호**와 PNG 파일명을 맞춘다.

## 9. 리스크·안전(필수 기재 권장)

- **인명**: 연못·호수 시험 시 **구명조끼·동승 제한·날씨** 준수.  
- **환경**: 종자(또는 대체 모의물) 투하가 **수질·생태**에 미치는 범위는 **지자체·소유자 허락** 범위 내에서만.  
- **전파·주파수**: LTE·Wi-Fi는 **해당 구역 사용 가능** 여부 확인.  
- **데이터**: 시험 좌표·함명이 **공개 부적절**하면 제출 전 **비식별화**.
