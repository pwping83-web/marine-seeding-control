# 프로젝트 기술 요구사항 정의서 (관공서 제출용 재구성)

본 문서는 **요구사항(목표)** 과 **현재 레포 구현**을 분리하여 기술적 정합성을 유지한다. 제출처에 따라 문구를 줄일 수 있다.

---

## 1. 초정밀 위치 측위 및 데이터 정합성 (RTK-GNSS)

### 1.1 요구사항 (목표)

- **정밀도:** cm급(예: ±5 cm 수준) 위치 확보를 목표로 할 수 있음(실증·사업 요구에 따름).
- **정합성:** 살포 1회와 관제 DB의 **좌표 1건**이 1:1로 대응하도록 채터링·중복 전송을 억제.

### 1.2 기술적 대응 방향 (설계·표준)

| 방향 | 설명 |
|------|------|
| RTK·NTRIP·VRS | 국토지리정보원 등 **NTRIP 캐스터**와 LTE로 보정 데이터를 받아 RTK 고정해(fixed) **cm급**을 노리는 구성은 **일반적인 해상·측량 설계**이다. |
| NMEA 0183 | `$GPGGA`, `$GPRMC`, `$GPVTG` 등에서 위·경도·고도·속도를 파싱하는 **표준 관행**이다. |
| 디바운스 | 물리 스위치의 바운스로 인한 **중복 트리거**를 소프트웨어 타이머로 제거하고, **최소 간격·쿨다운**으로 1회 살포당 1건 전송에 가깝게 만든다. |

### 1.3 현재 저장소 구현 상태

| 항목 | 상태 | 근거·비고 |
|------|------|-----------|
| RTK·NTRIP·VRS | **미구현(문서·스케치 TODO)** | Arduino 스케치는 UART에서 `$GPRMC`/`$GPGGA` **파싱을 사용자 TODO**로 두고, 시험용 고정 좌표 경로가 있다(`esp32_marine_telemetry*.ino`). **관공서 제출 시 “도입 예정·실증 과제”로 쓸지, “현 구현”으로 쓸지 반드시 구분.** |
| 웹 브라우저 위치 | **구현됨(일반 GNSS)** | `App.tsx` 등 `navigator.geolocation` — **RTK가 아님**. |
| NMEA 파싱(단말) | **부분(주석·PID 스니펫)** | `seeding-plan-ai.ts` 생성 스니펫에 `$GPVTG` 언급. 실제 펌웨어 파싱은 **현장 구현** 단계. |
| 디바운스·간격 제어 | **구현됨(상수 기준)** | `DROP_DEBOUNCE_MS` **50 ms**, `SEED_DROP_MIN_INTERVAL_MS` **800 ms**, `DROP_COOLDOWN_MS` **1500 ms** 등(`esp32_marine_telemetry.ino`, motor_relay 스케치 동계열). **500 ms**는 별도 튜닝 값으로 문서에 적을 수 있으나 **현 코드 기본값은 50 ms**임을 제출본에 맞출 것. |

## 2. 해상 특화 통신 아키텍처 (LTE & Edge Function)

### 2.1 요구사항 (목표)

- 해상 LTE **단절·지연**에도 운영 가능에 가깝게.
- 다수 선박 **동시 관제** 시 서버 과부하·오남용 방지.

### 2.2 기술적 대응 방향

| 방향 | 설명 |
|------|------|
| 이원화 엔드포인트 | **살포 이벤트**(burst 성격)와 **주기 궤적**(지속 스트림)을 **URL·페이로드 스키마**가 다른 Edge로 분리. |
| 레이트 리밋 | IP·글로벌 윈도우로 초당 요청 상한, 본문 크기·배열 길이 상한. |
| 오프라인 큐 | 단절 시 플래시/RAM에 적재 후 **타임스탬프 순 재전송** — **운영 권장** 패턴. |

### 2.3 현재 저장소 구현 상태

| 항목 | 상태 | 근거 |
|------|------|------|
| 이원화 Edge | **구현됨** | `supabase/functions/telemetry-ingest/index.ts`, `vessel-track-ingest/index.ts` |
| 장비 시크릿·HTTPS POST | **구현됨** | `X-Device-Ingest-Key`, `DEVICE_INGEST_SECRET` |
| 레이트 리밋(상이한 상수) | **구현됨** | 예: 살포 IP **25**/s·글로벌 **8000**/s, 궤적 IP **15**/s·글로벌 **4000**/s(코드 상수 기준). |
| 페이로드 검증 | **구현됨** | 좌표 범위, `recorded_at` 범위, 최대 건수·바이트. |
| 오프라인 큐(펌웨어) | **설계 권장·문서** | `arduino/아두이노-스케치-모음/문서/02-해상-통신-범위-한계.md` 등. **기본 스케치에 완전 구현으로 단정하지 말 것.** |

## 3. AI 기반 항행 보조 및 복원 면적 산출 (Algorithm)

### 3.1 요구사항 (목표)

- 살포 **효율적 항로 안내** 및 **조성·작업 면적** 설명 자료.
- 지도상 **시계열(연령)** 구분.

### 3.2 기술적 대응 방향

| 방향 | 설명 |
|------|------|
| A* (A-Star) | 암초·보호구역 등 **장애물 반경**을 회피하는 그리드 탐색. 스텝 비용은 **haversine 거리(km)**에 **풍향·풍속·돌풍 기반 배수**(역풍 페널티, 상한)를 곱한 값으로 할 수 있다. |
| Convex Hull 면적 | 살포 점 집합의 **볼록 껍질** + 면적 공식으로 **ha 근사**. |
| 시각화 | 위경도를 **지도(Leaflet 등)**에 표시하고 `recorded_at` 기반 **색·밴드**로 구분. |

### 3.3 현재 저장소 구현 상태

| 항목 | 상태 | 근거 |
|------|------|------|
| A* 장애물 회피 | **구현됨** | `src/lib/seeding-plan-ai.ts` — `findAStarRoute`, `KNOWN_OBSTACLES` |
| A* 스텝 기상 가중 | **구현됨** | 동일 파일 `weatherStepCostMultiplier` — 풍향·풍속·돌풍(역풍), `findAStarRoute(..., wx)` |
| 기상·한도·구역 점수 | **구현됨** | 동일 파일 `rankSeedingZones`, `calcDailyLimit` 등. A* 스텝 비용에는 **`weatherStepCostMultiplier`**로 풍향·풍속·돌풍이 **별도 반영**된다(파고는 스텝 비용에 미포함). |
| Convex Hull + ha | **구현됨** | `src/lib/seeding-day-eval.ts` — `estimateSeedingAreaHa`, 주석상 **참고치·비정밀 측량**. |
| 지도 렌더링 | **구현됨** | `MarineLeafletMap.tsx`, `Dashboard.tsx` 등 — **GeoJSON 파일 파이프라인으로 일원화된 표현은 필수 아님**(좌표·레이어 기반). |
| 연령(Age) 색상 | **구현됨** | `src/lib/seed-drop-visual.ts` — `dropAgeBand`, `seedDropMarkerColors` |

## 4. 제출 시 한 줄 가이드

- **“RTK ±5 cm”**는 **현 코드가 보장하지 않음** → 실증 계획·장비 납품 범위로 쓰거나, RTK 연동 후 **버전·날짜**를 명시한 문서만 제출.  
- **“500 ms 디바운스”**는 **설계 예시**로 두고, 실제 제출 수치는 **스케치 상수(50/800/1500 ms 등)**와 맞출 것.  
- **Convex Hull·이원 Edge·A\***는 **구현 근거가 명확** — `02-특허-실시예-명세-초안.md`와 함께 첨부하면 된다.
